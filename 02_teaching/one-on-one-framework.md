# 1-on-1 個別指導框架

## 基本規格

- 每週 1 小時，每位學生固定時段
- 目的：解卡點、確認進度、做題目與策略的個別調整

## 固定流程（每次 60 分鐘）

| 時段 | 時間 | 內容 |
|------|------|------|
| 回顧 | 10 分鐘 | 學生說上週做了什麼，老師確認成果 |
| 主要討論 | 35 分鐘 | 針對當週卡點、題目、程式或報告做深入討論 |
| 下週目標 | 10 分鐘 | 確認下週要完成什麼（具體、可驗收） |
| 策略備註 | 5 分鐘 | 若有升學、競賽、時程規劃，在此討論 |

## 初次 1-on-1 問題清單（了解學生背景）

1. 你現在用 Python 做過最複雜的事是什麼？
2. 你對哪些領域或問題最有興趣（不限於 CS）？
3. 你現在有沒有 AP/IB/A-Level 的課程壓力或 deadline？
4. 你希望在這個課程結束後，手上有什麼？
5. 你平常學習上最常卡在哪裡（語法？邏輯？還是不知道從哪下手）？

## 每週學生紀錄表欄位

| 欄位 | 說明 |
|------|------|
| 週次 | 本次是第幾週 |
| 目前專題題目 | 最新版本 |
| 目前 RQ | 最新版本（Level 2 以上） |
| 本週完成事項 | 學生回報 |
| 主要卡點 | 這週討論的核心問題 |
| 給的建議 | 老師給的具體建議 |
| 下週目標 | 具體、可驗收 |
| 備註 | 升學時程、家長溝通、特殊情況 |

建議用 Notion / Google Sheets 管理，每位學生一個分頁。

## 常見卡關點與處理方式

| 卡關類型 | 症狀 | 處理方式 |
|----------|------|----------|
| 題目太大／太抽象 | 說不清楚「我在模擬什麼」 | 縮小範圍：先做最簡單的版本，其他留到之後 |
| 模型太複雜 | 程式跑不動或結果不合理 | 簡化假設，先讓一個部分跑通 |
| 報告寫不出「討論與限制」 | 討論段落只是重複結果 | 問三個問題：為什麼是這個結果？這跟直覺一致嗎？如果模型改一個假設，結果會怎樣？ |
| 不知道怎麼找文獻 | 找不到相關 paper，或找到太多無關的 | 從最簡單的 Google Scholar 關鍵字開始，先找 1–2 篇讀懂，再從參考文獻延伸 |
| 程式 bug 解不掉 | 卡超過 30 分鐘 | 先寫最小可重現版本；教 print debugging 方式 |
