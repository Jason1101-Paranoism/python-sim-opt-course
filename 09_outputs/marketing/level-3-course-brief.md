# Level 3：數值方法與優化
### 讓你的專題在技術和數學上，達到大學初階課程的水平

---

## 一句話定位

從「能做專題」升級到「用 NumPy、SciPy 和最佳化方法，把專題做得更專業、更接近真實科學運算」。

---

## 這門課適合誰？

**適合：**
- 完成過 Level 2 或有完整 side project 經驗的學生
- 希望在技術層面有實質提升，不只是「會跑出結果」
- 準備 STEM 競賽（如數模、科展）、需要在程式與方法章節有更強內容的學生
- 對物理、工程、生態、金融等量化領域有興趣，想讓模型更接近真實的學生

**先修條件：**
- Level 2 完成，或有完整模擬專題基礎（能獨立設計與撰寫模擬程式）
- 不需要先修微積分或線性代數，但對數學有基本好奇心會更好

---

## 10 週學什麼

| 週次 | 主題 | 本週你會做到 |
|------|------|-------------|
| 第 1 週 | 目標設定與舊專題回顧 | 寫出「我的舊專題可以如何技術升級」的說明 |
| 第 2 週 | NumPy：陣列與向量化運算 | 將自己專題的關鍵計算改寫成 NumPy 版本 |
| 第 3 週 | SciPy：數值求根與積分 | 完成 1 個 SciPy 數值求根或積分程式 |
| 第 4 週 | SciPy：簡單 ODE 模型 | 用 ODE solver 模擬一個動態系統 + 時間演化圖 |
| 第 5 週 | 最佳化入門 | 用 SciPy optimize 找一個最大值或最小值 |
| 第 6 週 | 高級專題選題與設計 | 完成「高級專題計畫」一頁（含工具與方法選擇） |
| 第 7 週 | 專題實作：模型與數值方法 | 完成含數值方法的可執行專題程式 |
| 第 8 週 | 專題實作：最佳化與敏感度分析 | 完成最佳化流程 + 結果圖或表 |
| 第 9 週 | 技術寫作與簡報 | 完成「方法」與「實驗設計」兩段報告草稿 |
| 第 10 週 | 作品集展示與整理 | 完整程式碼 + 報告 + 簡報，整理成作品集 |

---

## 完課你會有什麼？

1. **完整高級專題**：含清楚方法章的技術報告，描述你使用的數值方法與工具
2. **可執行的專題程式**：使用 NumPy / SciPy，有向量化、ODE、最佳化其中至少一項
3. **研究簡報**：偏技術風格，可用於競賽展示或作品集
4. **整理好的作品集（portfolio）**：程式碼有結構、有注釋，附 README

**這份成果可以用在哪裡：**
- 科展、數模競賽的技術主體
- STEM 大學申請的 portfolio 核心作品
- 直接銜接 Level 4 的研究寫作

## 上課模式

| 項目 | 規格 |
|------|------|
| 課程長度 | 10 週 |
| 每週小班課 | 2 小時（4–8 人，固定時段） |
| 每週個別指導 | 1 小時 1-on-1（時段由學生與老師協商） |
| 每生總時數 | 30 小時 |
| 授課語言 | 中文為主，可依需求中英混合 |
| 上課地點 | 實體課（合作補習班）；特殊情況改線上 |

**1-on-1 個別指導** 每週都有，針對你的技術：
- 指導數值方法選擇與參數調整
- 協助把專題打磨成技術上說得過去的作品
- 討論作品集的整理與呈現方式
