# Level 3 週 05 作業單
## 作業名稱：SciPy Optimize 最佳化程式

---

## 作業目標

完成一個獨立的最佳化程式，使用 `scipy.optimize.minimize_scalar` 或 `scipy.optimize.minimize`（或兩者都用），解決一個有實際意義的「找最大／最小值」問題。

這個作業是 Level 3 Week 1–5 的整合：你要選一個問題，清楚定義目標函數、決策變數、限制條件，用 SciPy 求解，並用圖表展示最佳解。

## 具體要求

### 要求 1：選一個有實際意義的問題

**不接受**的題目：「找 f(x) = x^2 的最小值」（答案是 0，用微積分就知道了）

**可接受**的題目類型：

| 分類 | 例子 |
|------|------|
| 商業 / 經濟 | 最佳定價、廣告預算分配、庫存管理、最大利潤 |
| 工程 / 物理 | 最省能源的設計參數、結構的最小重量 |
| 數學 / 統計 | 最小二乘法擬合、找機率分布的最大概似估計 |
| 生物 / 生態 | 最佳收割策略（Logistic 模型）、族群管理 |
| 結合 ODE | 找讓 ODE 模型某個指標最優的參數（例如：最佳疫苗接種率） |
| 你的舊專題 | 在你的舊專題中加入最佳化步驟 |

### 要求 2：在程式碼頂端用注釋寫出問題框架

```python
# 問題框架
# 問題描述：[一兩句說明你要解決什麼問題]
# 目標函數：[你要最小化還是最大化什麼？數學形式]
# 決策變數：[哪些變數是你要找的？各自的意義和單位]
# 限制條件：[有哪些限制？或寫「無限制」]
# 工具：[你用了哪個 SciPy 函數，為什麼選它]
```

例如：
```python
# 問題框架
# 問題描述：找讓年度庫存總成本最低的訂貨量
# 目標函數：最小化 C(Q) = K*D/Q + h*c*Q/2（訂貨成本 + 持有成本）
# 決策變數：Q（每次訂貨量，單位：個）
# 限制條件：Q > 0
# 工具：minimize_scalar（method='bounded'），因為只有一個決策變數且有邊界
```

### 要求 3：程式必須包含以下部分

1. **問題框架注釋**（要求 2）
2. **目標函數定義**（有注釋說明每一項的含義）
3. **SciPy 最佳化求解**（並印出 `result.success`，確認成功）
4. **目標函數圖表**（一維：畫出函數曲線並標記最佳解；多維：畫出任一有意義的可視化）
5. **結果輸出**：印出最佳解和最佳目標函數值，並說明意義

### 要求 4：驗證最佳解

使用以下其中一種驗證方式：

**選項 A**：如果有解析解，與解析解比較
```python
print(f"數值解：x* = {result.x:.6f}")
print(f"解析解：x* = {analytical_solution:.6f}")
print(f"誤差：{abs(result.x - analytical_solution):.2e}")
```

**選項 B**：如果沒有解析解，用「鄰近點比較」驗證
```python
x_opt = result.x
delta = 0.01
print(f"最佳點 f({x_opt:.4f}) = {objective(x_opt):.6f}")
print(f"鄰近點 f({x_opt+delta:.4f}) = {objective(x_opt+delta):.6f}")
print(f"鄰近點 f({x_opt-delta:.4f}) = {objective(x_opt-delta):.6f}")
# 最佳點應該比兩個鄰近點都小（最小化問題）
```

**選項 C**：如果是多維問題，驗證限制條件是否成立
```python
x_opt = result.x
print(f"決策變數：{x_opt}")
print(f"限制條件驗證：{sum(x_opt):.4f}（應為 1.0）")
```

### 要求 5：結果說明

在程式末尾，用注釋或 Markdown 寫 2–3 句話：
1. 最佳解是多少（具體數字）
2. 這個結果在問題背景下的意義是什麼
3. 如果改變某個條件（例如成本增加、限制放寬），最佳解會如何變化？（可以口頭說明，不需要再跑一次）
