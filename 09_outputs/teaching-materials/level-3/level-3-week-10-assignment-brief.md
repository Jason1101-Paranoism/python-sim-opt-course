# Level 3 週 10 作業單：高級專題完整作品集

## 作業名稱與目標

**作業名稱**：Level 3 高級專題作品集

**目標**：
將這學期的高級專題整理成完整的作品集格式，包含可執行的程式碼、完整的技術報告草稿，以及整理好的展示簡報，使這份作品可以被外部人士（如大學教授、競賽評審、招生委員）閱讀和理解。

---

## 繳交清單

| 項目 | 格式 | 說明 |
|------|------|------|
| 主程式 | `.py` 檔 | 可執行，有中文注釋和 docstring |
| README | `.md` 檔 | 說明專題、方法、如何執行、主要結果 |
| requirements.txt | 文字檔 | 列出所有依賴套件及版本 |
| 輸出圖表 | 資料夾內的 `.png` 檔 | 至少：模型輸出圖、收斂測試圖、最佳化結果圖、敏感度分析圖 |
| 技術報告草稿 | `.md` 或 `.docx` | 含：摘要、問題、模型、方法、結果、限制六段 |
| 展示簡報最終版 | Google Slides / PPT / PDF | 根據今日展示回饋修改後的版本 |

**截止時間**：Level 3 期末展示後 **1 週內**

---

## 程式碼整理具體要求

1. **每個函數必須有 docstring**：說明功能、輸入參數（型別、含義）、回傳值
2. **主要段落需有中文注釋標題**：使用 `# === 段落名稱 ===` 的格式區隔各段
3. **不能有死碼（dead code）**：刪除所有被 `#` 注釋掉的舊程式碼、測試用的臨時片段
4. **程式從頭執行一次不報錯**：在乾淨環境（重新安裝套件後）確認可以跑
5. **輸出圖表自動存檔**：執行主程式後，圖表應該自動儲存到 `outputs/` 資料夾

---

## 技術報告草稿結構要求

| 章節 | 最低字數 | 必要內容 |
|------|---------|---------|
| 摘要 | 100 字 | 問題、方法、主要結果（一段話） |
| 問題陳述 | 150 字 | RQ、假設、研究範圍 |
| 模型 | 200 字 | 方程式或架構、狀態變數定義、參數含義 |
| 方法 | 200 字 | 工具（具體函數名）、關鍵參數、選擇理由 |
| 結果 | 200 字 | 主要發現（含數字）、最佳化結果、敏感度分析摘要 |
| 限制與未來方向 | 150 字 | 至少 2 個技術限制；至少 1 個未來延伸方向 |

## 自我檢查清單

### 程式碼

- [ ] 每個函數有 docstring（說明功能、參數、回傳值）
- [ ] 每個主要段落有中文注釋標題
- [ ] 沒有被注釋掉的死碼
- [ ] 在乾淨環境可以從頭執行不報錯
- [ ] 圖表輸出自動存到 `outputs/` 資料夾

### README

- [ ] 一段話說明這個專題是什麼
- [ ] 說明研究問題（RQ）
- [ ] 列出使用的工具（函數名稱）
- [ ] 說明如何執行（具體指令）
- [ ] 說明環境需求（Python 版本、套件版本）
- [ ] 至少一個技術限制

### 技術報告

- [ ] 包含六個章節（摘要/問題/模型/方法/結果/限制）
- [ ] 方法段有具體函數名稱和選擇理由
- [ ] 結果段有數字（不只有圖）
- [ ] 限制段有具體情境（不只說「假設不完美」）

### 展示簡報

- [ ] 根據今日展示的回饋修改過
- [ ] 10–12 張投影片
- [ ] 有收斂測試投影片
- [ ] 有敏感度分析投影片
- [ ] 限制投影片至少 2 個具體限制

## 作品集可以用在哪裡？

整理好的作品集可以用於：

**大學申請**：
- 備審資料中「自主學習成果」
- 說明你有獨立進行計算科學研究的能力

**科學展覽 / 競賽**：
- 台灣全國中小學科學展覽
- 全國科學探究競賽
- 國際科學展覽（ISEF）的本土選拔

**個人 GitHub**：
- 上傳為公開 repository，作為技術作品集
- 聯繫大學研究室時可附上連結
