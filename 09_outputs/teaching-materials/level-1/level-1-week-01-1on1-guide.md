# Level 1 週 1：1-on-1 討論指南

---

## 本週 1-on-1 特殊目的

第一週的 1-on-1 是**了解學生**的最重要機會，優先於討論作業。

**目標**：
1. 了解學生的 Python 背景與學習動機
2. 初步確認學生能跑起來程式（環境 OK）
3. 種下「這是你自己的專題課」的期待

---

## 流程（60 分鐘）

| 段落 | 時間 | 內容 |
|------|------|------|
| 學生背景了解 | 20 分鐘 | 初次訪談問題清單 |
| 作業回顧 | 15 分鐘 | 看學生的骰子程式，確認能跑、能解釋 |
| 環境確認 | 10 分鐘 | Python 環境、編輯器、matplotlib 安裝 |
| 下週目標設定 | 10 分鐘 | 確認學生對下週機率遊戲的初步想法 |
| 備用（雜問） | 5 分鐘 | 學生提問、課程說明補充 |

---

## 初次訪談問題清單（前 20 分鐘）

> 目的：了解學生，調整後續 1-on-1 的深度與方向。不是考試，學生說什麼都 OK。

**Python 背景**
- 「你之前用 Python 做過最複雜的事是什麼？」
- 「你比較習慣用哪個編輯器？（VS Code / Jupyter / 其他）」
- 「for 迴圈、函數、list 這三個你都熟嗎？哪個最不熟？」

**學習動機與目標**
- 「你來這個課程最想學到什麼？」
- 「你對哪些主題或領域最有興趣？（不限 CS，可以是物理、社會、遊戲）」
- 「你有沒有特別想做的專題？或者完全沒想法也 OK」

**升學與課外壓力**
- 「你現在幾年級？學校有 AP/IB/A-Level 的課嗎？」
- 「段考大概幾月？我們可以把最難的作業排開那段時間」

**記錄在學生紀錄表**（Notion/Sheet）：Python 程度評估、興趣方向、升學需求、特殊限制

---

## 作業回顧（第 15 分鐘）

**請學生分享螢幕，展示骰子程式**

確認以下三點：
1. **程式能跑**：`python level1-week01-xxx.py` 沒有 error
2. **圖表有基本元素**：標題、軸標籤
3. **學生能解釋**：「為什麼用 `randint(1, 6)`？」「`range=(0.5, 6.5)` 是什麼意思？」

**常見卡點處理**：

| 卡點 | 建議處理 |
|------|----------|
| matplotlib 沒安裝 | 現場一起裝：`pip install matplotlib`；若有權限問題另行處理 |
| 程式有 error 但不知道為什麼 | 一起讀 error message 的最後一行，教「從下往上讀」的習慣 |
| 圖畫出來但 bins 不對 | 解釋 `bins` 和 `range` 的關係，讓學生自己調整 |
| 學生說「我不會 matplotlib，但程式跑了」 | 接受；matplotlib 不是本週重點，下週會再深入 |

## 下週目標設定（第 10 分鐘）

告訴學生下週主題（彩票 / 生日悖論），問：

- 「你對彩票還是生日悖論比較有興趣？」（下週可以讓學生選一個做）
- 「你這週的骰子程式，有沒有想再改進的地方？」

**確定下週里程碑**（具體、可驗收）：
> 下週 1-on-1 前，學生要有：選好題目（彩票或生日悖論）+ 第一版可執行程式

---

## 本週學生里程碑

- [ ] 骰子模擬程式可執行
- [ ] 直方圖輸出有標題與軸標籤
- [ ] 學生能用 1–2 句話解釋程式在做什麼
- [ ] 環境正常（Python + matplotlib 可用）
- [ ] 學生紀錄表已建立並填入背景資訊

---

## 老師備忘

**第一週常見情況**：
- 有些學生程式很強，但對「為什麼要模擬」完全沒概念 → 花時間在概念上
- 有些學生概念懂，但 Python 很生疏 → 確保環境正常，作業量可以先縮小
- 有些學生非常緊張（第一次上這種課） → 反覆強調「不打分、有問題可以 1-on-1 問」
