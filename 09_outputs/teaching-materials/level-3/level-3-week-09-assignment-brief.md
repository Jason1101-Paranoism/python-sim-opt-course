# Level 3 週 09 作業單：報告草稿 + 期末展示簡報初稿

## 作業名稱與目標

**作業名稱**：技術報告草稿 + 展示簡報初稿

**目標**：
完成專題報告的「方法」與「實驗設計」兩段書面草稿，並完成期末展示簡報的初稿（10–12 張投影片）。這兩份材料是下週期末展示的基礎。

---

## 繳交清單

| 項目 | 格式 | 說明 |
|------|------|------|
| 方法段草稿 | `.md`、`.docx` 或 Google Doc | 150–250 字 |
| 實驗設計段草稿 | （同上，可合在一份文件） | 100–200 字 |
| 展示簡報初稿 | Google Slides、PowerPoint 或 Keynote | 10–12 張，包含所有必要投影片 |

**截止時間**：下次 group 課前（週 10 上課前 48 小時）

> 老師會在展示前提供書面或口頭回饋。請確保在期末展示前 48 小時繳交，以便老師能讀完。

---

## 方法段格式要求

方法段必須包含：

1. **模型說明**（1–3 句）：描述你的模型（方程式類型、狀態變數），可以包含方程式
2. **工具說明**（1–2 句）：明確說出使用的 Python 函數名稱（如 `scipy.integrate.solve_ivp`）
3. **參數設定**（1–2 句）：說明關鍵參數（步長、容忍度、搜尋範圍等）是如何選定的
4. **工具選擇理由**（1–2 句）：說明為什麼選這個方法，而不是最簡單的替代方案
5. **技術限制**（2–3 句）：至少提到一個模型假設的限制和一個數值方法的限制

---

## 實驗設計段格式要求

實驗設計段必須包含：

1. **基準設定**：說明基準實驗的參數值（「基準條件為 β=0.3, γ=0.1, N=10000」）
2. **實驗組設計**：說明你改變了哪些條件（每組實驗的控制變數）
3. **觀察指標**：說明你要觀察什麼輸出（「觀察感染高峰人數和達到高峰的時間」）
4. **比較目標**：說明各組實驗之間要比較什麼（「比較不同 β 值下的最佳接種策略」）

---

## 展示簡報格式要求

| 投影片 | 必要內容 |
|--------|----------|
| 1：標題頁 | 題目、姓名、日期 |
| 2：問題陳述 | 你要解決什麼問題（一句話 RQ） |
| 3–4：模型 | 核心方程式或架構；假設列表（至少 2 個） |
| 5：方法 | 工具名稱、關鍵參數、選擇理由 |
| 6：收斂測試 | 一張收斂測試圖，說明你確認了數值精度 |
| 7–8：主要結果 | 模型輸出圖 + 最佳化結果（含數字） |
| 9：敏感度分析 | 一張敏感度分析圖，一句話說明趨勢 |
| 10：限制與延伸 | 至少 2 個技術限制；未來可以做什麼 |

---

## 具體要求

1. **方法段不得出現程式操作語言**：不能寫「先 import，然後呼叫函數，最後 print 結果」
2. **工具名稱必須具體**：說「SciPy 的 `scipy.integrate.solve_ivp`（RK45 方法）」，不只說「SciPy」
3. **限制必須有具體情境**：「假設均勻混合，若實際人群有社交聚集，結果可能低估局部傳播」（這種程度）
4. **簡報不得有超過 5 行文字的投影片**：圖表為主，文字為輔
5. **結果投影片必須有數字**：「最佳接種比例為 67%」比「最佳化找到了好的解」更有說服力

---

## 自我檢查清單

### 方法段 + 實驗設計段

- [ ] 方法段有提到具體的函數名稱（不只說「用 SciPy」）
- [ ] 方法段說明了關鍵參數的選擇理由（不只說「設為 X」，要說「為什麼設為 X」）
- [ ] 方法段有至少 2 個技術限制（模型假設 + 數值方法）
- [ ] 實驗設計段有「固定 X，改變 Y，觀察 Z」的結構
- [ ] 兩段都沒有程式操作語言（不是操作手冊）
- [ ] 字數符合要求（方法段 150–250 字，實驗設計段 100–200 字）

### 展示簡報

- [ ] 有 10–12 張投影片，涵蓋所有必要段落
- [ ] 每張投影片不超過 5 行文字
- [ ] 有收斂測試圖（第 6 張）
- [ ] 有敏感度分析圖（第 9 張）
- [ ] 限制投影片有至少 2 個具體限制（不是泛泛而談）
- [ ] 整份簡報 6 分鐘可以講完（可以默唸計時）

## 常見失分項目

| 問題 | 說明 |
|------|------|
| 方法段只有「做了什麼」，沒有「為什麼」 | 每一個工具選擇都要附上理由 |
| 限制段只說「模型不夠完美」 | 限制要具體：在什麼條件下、哪個假設、會造成什麼方向的偏差 |
| 簡報沒有收斂測試投影片 | Level 3 的技術誠信要求展示你確認了數值精度 |
| 結果投影片沒有數字 | 圖表上要有具體數字（如最佳解的值），不能只有圖形 |
