# 學生入班清單

> 本清單適用於**確認入班後、第一堂課前**的所有準備工作。
> 分為兩份：「老師端清單」與「學生 / 家長端清單」。
> 建議在開課前 **5–7 天**完成所有項目。

---

## 一、老師端清單

### 行政確認

- [ ] 確認報名表已收到（`intake-form-spec.md`）
- [ ] 確認繳費已完成（由補習班行政確認，或直接轉帳確認）
- [ ] 確認同意書與隱私授權已簽署（`consent-and-ip-policy.md`）
- [ ] 在學生管理表（Google Sheets / Notion）中建立此學生的個人頁
- [ ] 排定第一次 1-on-1 時段（於 group 課第一週後進行）
- [ ] 確認學生的偏好時段與 group 課時段沒有衝突

### 教材準備

- [ ] 確認學生的入班 Level，準備對應的第一週教材
- [ ] 準備「學生環境確認清單」（見下方學生清單第二節），傳送給學生
- [ ] 若學生為直接入班，複查程度測驗紀錄，標記可能的薄弱點

### 家長溝通

- [ ] 寄送「課程說明文件」給家長（課程結構、每週時數、作業量、溝通管道）
- [ ] 說明 1-on-1 排定方式，請家長或學生提供可用時段
- [ ] 確認家長的主要聯絡管道（LINE / Email）

---

## 二、學生 / 家長端清單

> 以下清單會以文字訊息或 Email 傳送給學生，請在第一堂課前完成。

### 第一節：確認你的設備環境

- [ ] 已在電腦安裝 Python 3.10 以上版本（建議使用 Anaconda / Miniconda）
- [ ] 可以在終端機 / Jupyter Notebook 中成功執行 `print("Hello World")`
- [ ] 已安裝以下套件（依入班 Level）：
  - Level 1–2：`matplotlib`, `numpy`（Anaconda 預裝，一般不需另行安裝）
  - Level 3：另需確認 `scipy` 可用（執行 `import scipy` 無錯誤）
  - Level 4：Level 3 套件 + 確認可開啟 PDF 文件進行閱讀與筆記
- [ ] 電腦螢幕可以在上課時分享（Google Meet / Zoom）

### 第二節：閱讀與簽署文件

- [ ] 已閱讀「課程說明文件」（包含上課規則、作業政策、退費政策摘要）
- [ ] 已完成「同意書與隱私授權」簽署（`consent-and-ip-policy.md` 對應表單）

### 第三節：準備第一堂課

- [ ] 知道 group 課的時間、地點（或線上連結）
- [ ] 知道第一次 1-on-1 的排定時間
- [ ] 有一個初步想做的題目方向（不需確定，有大致興趣即可）
  > 提示：你對哪些現實問題有好奇心？可以是物理、生態、交通、社會、運動等——任何可以「用數字描述」的事情都可以。
- [ ] 準備好紙筆或筆記工具（每週課堂會有討論和手寫的部分）

---

## 第一次 1-on-1 的老師準備

> 第一次 1-on-1 在第一週 group 課後進行（不等第二週）。

**時長**：45–60 分鐘

**目標**：
1. 了解學生的背景、學習動機與時間狀況
2. 確認環境設定正常（當場測試）
3. 開始收斂題目方向（Level 1：從老師建議清單選擇；Level 2+：學生提出初步想法）
4. 說明 1-on-1 的互動方式（不直接給答案，引導學生思考）

**老師提問清單（第一次 1-on-1）**：

- 「你以前有沒有做過讓你覺得有趣的 Python 程式？」
- 「這學期課業壓力大嗎？每週大概可以花多少時間在這個課的作業？」
- 「你知道 Level [X] 期末要做出什麼嗎？你有什麼期待？」
- 「有沒有什麼你怕遇到的困難？」（幫助老師預先了解可能的卡關點）

## 特殊情況處理

| 情況 | 處理方式 |
|------|---------|
| 開課前 3 天才確認入班 | 縮短 checklist，至少確保環境設定 + 同意書完成 |
| 學生設備安裝有問題 | 第一堂課前安排 15 分鐘 Meet 協助排除（或建議改用 Google Colab 暫代） |
| 家長要求延後第一次 1-on-1 | 可延至第 2 週，但需提前告知 |
