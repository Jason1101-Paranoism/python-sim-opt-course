# Level 3 週 01 學生講義
## 主題：目標與舊專題回顧

---

## 本週你要做什麼

Level 3 的目標不是從零開始學新東西，而是把你已經有的專題「升級」——用更專業的工具、更嚴謹的方法，讓程式跑得更快、結果更可信、報告更有說服力。

本週你要做的事：
1. 重新看一次自己的舊專題
2. 找出「可以升級」的地方
3. 寫下你的升級計畫

---

## 核心概念一：什麼是「技術升級」？

**升級不是重寫，是提升**。

你在 Level 2 已經能寫出一個可以跑的專題。Level 3 要做的，是把它從「可以跑」提升到「有方法論」。具體來說：

| 升級前（Level 2）| 升級後（Level 3）|
|-----------------|-----------------|
| 用 for 迴圈做大量計算 | 改用 NumPy 向量化，速度提升數十到數百倍 |
| 用手動嘗試找參數 | 用 SciPy optimize 正式找最佳解 |
| 用差分方程近似動態系統 | 用 SciPy ODE solver 精確求解 |
| 報告寫「用 Python 計算」 | 報告寫「使用 `scipy.optimize.minimize` 求解目標函數」 |

### 範例：同一個計算，兩種寫法

```python
# ───── 升級前（Level 2 風格） ─────
results = []
for i in range(10000):
    results.append(i * 2.5 + 1.3)

# ───── 升級後（Level 3 NumPy 風格） ─────
import numpy as np
i = np.arange(10000)
results = i * 2.5 + 1.3   # 同樣結果，但快 100 倍以上
```

這兩段程式的計算結果完全相同，但 NumPy 版本：
- **速度更快**：向量化運算省略了 Python 迴圈的開銷
- **程式更短**：邏輯清楚、不容易出錯
- **更接近數學**：`i * 2.5 + 1.3` 就是數學式

## 核心概念二：Level 3 工具地圖

本課程會用到的主要工具：

```
你的專題問題
      │
      ├── 大量數值計算？          →  NumPy（週 2）
      │
      ├── 需要解方程或積分？      →  SciPy optimize / integrate（週 3）
      │
      ├── 模擬動態系統（隨時間變化）？ →  SciPy odeint（週 4）
      │
      └── 想找最佳參數？          →  SciPy minimize（週 5）
```

**重要**：這些工具不是獨立的，你的期末專題很可能同時用到 2–3 個。

## 核心概念三：Level 3 的完整成果長什麼樣

期末（週 10）你要交出：
- **程式碼**：結構清楚、有註解、可以直接執行
- **報告**：含「方法」章節，清楚描述你用了哪些數值方法、為什麼選這個方法
- **簡報**：能在 10 分鐘內說明你的問題、方法和發現
