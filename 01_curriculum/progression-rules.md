# 升降級與入班規則

---

## 標準升級路徑

```
Level 1 → Level 2 → Level 3 → Level 4
```

每一級完成後，預設可直接升下一級，無需額外測驗。

---

## 直接入班規則（跳過某一 Level）

### 直接入 Level 2（跳過 Level 1）

**條件**（需同時符合）：
- 通過程度測驗 Part A（Python 基礎）+ Part B（模擬概念）
- 能說明「模擬 vs 直接計算的差異」
- 曾自己做過一個有圖表輸出的 Python 程式（提供截圖或程式碼）

**程度測驗分流標準**：
- Part A（Python 基礎）≥ 70 分
- Part B（模擬概念）≥ 60 分

### 直接入 Level 3（跳過 Level 1–2）

**條件**（需同時符合）：
- 通過程度測驗 Part A + B + C（數值方法基礎概念）
- 有一個完整的 side project 或 AP/IB/A-Level 專題成果可供審閱
- 說得出自己模型的假設與限制

### 直接入 Level 4（跳過 Level 1–3）

**條件**（需同時符合）：
- 通過程度測驗 Part A + B + C + D（文獻閱讀概念）
- 有使用 NumPy 或 SciPy 的程式作品
- 有初步閱讀過 1 篇以上 STEM 相關論文的經驗
- 1-on-1 諮詢後老師確認適合

> **注意**：直接入 Level 3 或 4 的案例較罕見，通常發生在有大學 RA 或研究室經驗的學生。

---

## 程度測驗結構（分四部分）

| 部分 | 對應入班門檻 | 題型 |
|------|-------------|------|
| Part A：Python 基礎 | 所有 Level 均需通過 | 選擇題 + 讀程式填空（10 題） |
| Part B：模擬概念 | 直接入 Level 2 | 情境題（3 題），說明如何把問題變成模擬 |
| Part C：數值方法概念 | 直接入 Level 3 | 選擇題 + 簡答（5 題） |
| Part D：研究流程概念 | 直接入 Level 4 | 閱讀一段論文摘要，回答 3 個問題 |

## 降級規則

### 觸發條件

以下任一情況發生，老師需在 1-on-1 中討論是否降級：

1. **連續兩週作業未達最低要求**（rubric 中三項以上為「待改」）
2. **中期展示時，程式無法執行或完全沒有產出**
3. **學生本人明確表示壓力過大，想從基礎開始**

### 降級流程

1. 老師在 1-on-1 提出觀察（不當眾說）
2. 說明降級的好處：「先把基礎做紮實，之後進步會更快」
3. 通知家長（說明原因與預期效益）
4. 辦理課程轉班（依退費轉班政策處理）

### 降級不適用的情況

- 學生因個人因素（考試週、生病）短暫落後 → 補課或放慢進度處理
- 學生程式弱但模型思維強 → 加強 1-on-1 debug 時間，不降級

---

## 同一 Level 重修規則

**適用情況**：學生完成某 Level 但期末成果明顯不完整，或想把專題做得更好。

**規定**：
- 可以重修同一 Level，費用按當期定價計算
- 重修學生的 1-on-1 聚焦在「上次沒完成的部分」
- 重修不影響之後升級的資格

---

## 轉班規則

**同 Level 不同梯次轉班**：
- 開課前 7 天內提出，可免費轉至下一梯次（同 Level）
- 開課後轉班視同退費後重新報名，依退費政策計算

**跨 Level 轉班**（升班 / 降班）：
- 依降級流程或直接入班規則重新評估
- 費用差額多退少補

---

## 邊界案例處理

| 情況 | 處理方式 |
|------|----------|
| 學生通過程度測驗但老師評估程度不足 | 老師的評估優先，建議先上下一個 Level 確認 |
| 學生測驗分數略低（差 5 分以內）但有強烈動機 | 進行 1-on-1 諮詢後，由老師決定是否破例 |
| 家長希望學生跳級但學生本人不確定 | 以學生意願為主，家長需簽同意書 |
| 同一學生同時報名兩個不相鄰的 Level | 不建議；若堅持，老師需確認兩個專題不互相干擾 |
| 國際學生英文為主、中文程度有限 | Level 1–2 可提供中英混合講解；Level 3–4 建議切換至英文授課版 |
