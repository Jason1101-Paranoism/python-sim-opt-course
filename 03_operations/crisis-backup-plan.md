# 危機應變與備援計畫

---

## 情境一：老師臨時無法授課（生病 / 緊急事故）

### 發生條件

課程開始前 **6 小時內**（或已開課後）才確認無法上課。

### 應變步驟

1. **立即通知**：群發訊息給所有學生與家長，說明當日停課
2. **補課方式選擇**（依優先順序）：
   - **方案 A**：發佈預錄課程影片（事先準備好的備用教學影片）+ 發布非同步作業
   - **方案 B**：協商補課時間（課程週期內）
   - **方案 C**：若無法補課，退還該週比例費用（課程總費 ÷ 10）
3. **通知補習班行政**：同步告知教室可釋放

### 備用教學影片準備原則

- 每個 Level 至少準備 **2 週**的備用影片（建議為核心概念週次）
- 影片格式：螢幕錄影 + 語音解說，時長 40–60 分鐘
- 搭配當週練習題（學生自行完成，下週 group 課初統一討論）

---

## 情境二：學生人數不足，無法開班

### 觸發條件

開課前 **1 週**確認報名人數 **低於 3 人**。

### 應變步驟

1. **通知已報名學生**：說明開課條件不足，提供以下選項：
   - **選項 A**：等待下一梯次（費用保留，不退費）
   - **選項 B**：全額退費（依退費政策，無手續費）
   - **選項 C**：以「私人家教模式」開課（1-on-1 only，費用另議）
2. **繼續招生**（若距開課仍有時間）：通知補習班加強推廣
3. **記錄原因**：梯次結束後記錄招生不足的可能原因

### 私人家教模式費用參考

- 僅 1-on-1 課程，無 group 課
- 建議費率：**原課程費 × 1.3**（反映老師時間成本提高）
- 需學生/家長明確同意後才適用

## 情境三：實體課改線上（突發事故 / 疫情 / 天災）

### 觸發條件

教室無法使用、或公共衛生 / 安全因素需居家上課。

### 應變步驟

1. **通知學生與家長**：說明改為線上，提供會議連結
2. **線上工具**（建議）：
   - 視訊：Google Meet / Zoom（補習班現有系統為主）
   - 程式共享：VS Code Live Share 或 Google Colab
   - 白板：Miro / Google Jamboard
3. **課程結構維持不變**（見 `class-structure.md`），不因線上而縮短
4. **技術問題備案**：
   - 若學生視訊異常 → 改純語音 + 螢幕分享
   - 若多人斷線 → 暫停 5 分鐘等候，超過 15 分鐘則改非同步
5. **記錄**：通知補習班行政，教室費用協商退款或保留

### 長期線上課調整

若線上課持續超過 **2 週**：

- 1-on-1 改為純視訊，保持固定時段
- 調整作業繳交方式（GitHub / Google Classroom）

## 情境四：學生發生心理或行為危機

### 觸發跡象

- 學生多次表現明顯情緒低落、退縮
- 提及家庭壓力、心理健康問題
- 在課堂中突然離開或無故不回應

### 應變步驟

1. **不在群體中處理**：私下（1-on-1）關心
2. **傾聽優先**，不馬上給學業建議
3. **通知家長**：當天或次日（說「我有觀察到一些情況想和您說」，不過度渲染）
4. **調整課程**：降低當週作業要求，給學生喘息空間
5. **不超過能力範圍**：課程方非諮商機構，遇嚴重情況建議家長尋求專業協助

---

## 情境五：學生與其他學生發生衝突（Group 課）

### 觸發條件

- 同學間言語衝突
- 作品被抄襲投訴

### 應變步驟

1. **Group 課中立即降溫**：換話題或給個人實作時間
2. **課後個別談**：分別與涉及雙方溝通，不讓雙方對峙
3. **抄襲投訴**：
   - 比對程式碼相似度
   - 分別在 1-on-1 詢問（不公開指控）
   - 確認後：要求重做，並通知家長
4. **無法共存**：考慮協商梯次轉班

## 平時備援準備清單

每個梯次開課前確認：

- [ ] 備用教學影片已準備（至少 2 週）
- [ ] 線上授課工具帳號已設定（Zoom / Meet）
- [ ] 補習班行政聯絡方式已確認
- [ ] 退費計算表格已備好（依定價計算各週退費金額）
- [ ] 學生與家長 LINE 群組已建立
