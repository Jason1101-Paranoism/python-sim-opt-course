# Level 1 週 5：小專題 1 — 個人 Mini Project
## 教師教案（Teacher Notes）

---

### 本週學習目標

1. 學生能把自己選定的題目拆解成「狀態 + 規則 + 輸出」
2. 學生能完成第一版可執行的模擬程式（即使很簡陋）
3. 學生能說明程式在模擬什麼（不是只會跑）

---

### 課堂時間分配（120 分鐘）— 工作坊模式

| 段落 | 時間 | 內容 |
|------|------|------|
| 暖身與題目確認 | 15 分鐘 | 每人 1 分鐘說自己的題目，老師快速確認可行性 |
| 概念複習 | 15 分鐘 | 快速複習前四週的三個模板（機率、排隊、差分） |
| 個人實作 | 70 分鐘 | 學生動手做；老師巡視個別指導 |
| 回顧與預告 | 20 分鐘 | 每人分享進度（1 分鐘）、作業說明、預告視覺化 |

> 本週是工作坊模式，取消固定的概念講解與示範段落，延長實作時間。

---

### 暖身與題目確認（15 分鐘）

**快速一輪**：每人用 1 句話說「我要模擬什麼」。

老師快速判斷可行性：

| 狀況 | 處理方式 |
|------|----------|
| 題目合適（有隨機 + 有規則 + 可量化） | 給個「讚」，讓他繼續 |
| 題目太大（例：「模擬整個股市」） | 立刻縮小：「先只做一支股票的隨機漲跌，其他之後再加」 |
| 題目沒有隨機性（例：「計算距離」） | 引導加入隨機：「如果有誤差或不確定性，怎麼加進去？」 |
| 學生還沒想好 | 給三個預設選項（見 1-on-1 指南），讓他馬上選一個 |

---

### 概念快速複習（15 分鐘）

把前四週的三個模板整理在白板上，讓學生對號入座：

**模板 A：機率 / 隨機事件類**
```python
def simulate_once(params):
    result = random.某函數(...)
    return result

results = [simulate_once(params) for _ in range(N)]
# 畫直方圖或折線圖
```

**模板 B：排隊 / 時間序列事件類**
```python
current_time = 0
server_state = ...
for 每個事件:
    事件時間 = 目前時間 + 隨機間隔
    更新狀態
    記錄結果
```

**模板 C：差分 / 動態系統類**
```python
state = [initial_value]
for t in range(steps):
    next_state = state[-1] + 變化規則(state[-1])
    state.append(next_state)
# 畫折線圖
```

### 個人實作（70 分鐘）

**老師巡視重點**：
- 第 0–15 分鐘：確認每個學生有開始寫（有程式框架）
- 第 15–50 分鐘：個別指導，優先處理「程式完全沒有進展」的學生
- 第 50–70 分鐘：確認每個學生至少跑出一個結果（哪怕很粗糙）

**引導問題**（遇到卡關時用）：
- 「你的模擬裡，什麼東西是隨機的？」
- 「你想觀察的輸出是什麼？一個數字、還是一個時間序列？」
- 「先寫一個 `simulate_once()`，只做最簡單的一件事，能跑就好。」

---

### 回顧（20 分鐘）

每人用 1 分鐘分享：「我在模擬什麼，我現在跑出了什麼結果。」

**老師不評分，只問**：
- 「你的程式跑得出東西了嗎？」
- 「下週（視覺化週）你想讓這個程式多畫什麼圖？」

---

### 常見狀況

| 狀況 | 處理 |
|------|------|
| 學生做了和範例幾乎一樣的東西 | 接受；下週 1-on-1 再引導他加入個人特色 |
| 學生程式很複雜但跑不起來 | 幫他砍到最小可行版本：「先只做最核心的一步」 |
| 學生寫得很快，已經有結果 | 給他「加入隨機性」或「掃描一個參數」的挑戰 |
