# Level 3 週 10：作品集展示與整理

## 本週是什麼？

這是 Level 3 的最後一週，也是你整個高級專題的正式亮相。

本週有兩件事：

**1. 期末展示（課堂）**：5–8 分鐘，向全班和老師展示你的完整高級專題

**2. 作品集整理（課後）**：把你的程式碼、報告和簡報整理成「可以給別人看」的格式

---

## 期末展示指南

### 你的展示時間：5–8 分鐘

超過 8 分鐘老師會打斷你，請練習到 6–7 分鐘左右。

### 展示的技術深度（Level 3 標準）

這次展示的受眾是「有數學/科學背景的人」，而不是一般大眾。

**你必須說到的六件事**：

| 必要內容 | 範例說法 |
|----------|----------|
| 你的研究問題 | 「我研究的問題是：在傳染病爆發初期，最佳接種比例是多少？」 |
| 你的模型 | 「我使用 SIR 微分方程組，包含三個狀態：易感者 S、感染者 I、康復者 R」 |
| 你使用的工具 | 「數值求解使用 `scipy.integrate.solve_ivp`（RK45 方法），最佳化使用 `minimize_scalar`」 |
| 你如何驗證數值精度 | 「我做了收斂測試：max_step 從 5 天縮小到 0.05 天，高峰人數的誤差從 8% 降到 0.05%」 |
| 你的主要結果 | 「在我的基準條件下，最佳接種比例為 67%，對應感染高峰為 285 人（佔總人口 2.85%）」 |
| 你的技術限制 | 「最主要的限制是均勻混合假設——真實疫情中社交聚集效應會導致局部傳播遠高於模型預測」 |

---

### 展示時間分配建議

```
0:00–0:30  問題陳述（一句話說清楚你的 RQ）
0:30–2:00  模型與方法（方程式 + 工具名稱 + 收斂測試）
2:00–4:30  主要結果（模型輸出圖 + 最佳化結果 + 敏感度分析）
4:30–5:30  技術限制（2–3 個，具體）
5:30–6:00  未來延伸（可以做什麼下一步）
```

---

### 展示後的技術問答

展示結束後，老師會問 2–3 個技術問題。以下是最常見的問題，請準備好答案：

**問題 1：「你的 max_step 是怎麼選的？」**

準備好回答：「我跑了收斂測試，用 [幾個步長值] 比較，在 [某個步長] 以下，結果變化小於 [百分比]，所以選了 [你用的步長]。」

**問題 2：「你的最佳解是局部解還是全局解？」**

準備好回答：「我試了 [幾個] 不同的初始點，結果 [相同/相近]，所以 [很可能是全局解 / 有多個局部解，我取了最小的一個]。」

**問題 3：「你的模型最重要的假設是什麼，如果這個假設不成立，結論怎麼變？」**

準備好回答：說出你最關鍵的假設（如均勻混合、固定參數），說清楚違反假設時結果會往哪個方向偏差。

**問題 4：「敏感度分析中，哪個參數影響最大？」**

準備好回答：看你的敏感度分析圖，哪條曲線斜率最大（最敏感）。

## 作品集整理指南（課後完成）

Level 3 結束後，你有一份完整的高級專題。現在要把它整理成「作品集（portfolio）格式」。

### 什麼是作品集格式？

作品集格式的意思是：**一個陌生人在不問你的情況下，可以看懂你的專題是什麼、你做了什麼、程式怎麼跑。**

---

### 程式碼整理：四個要求

#### 要求 1：清楚的檔案結構

```
your_project/
│
├── README.md              ← 必須有（說明這個專題是什麼、怎麼執行）
├── main.py                ← 主程式（執行這個就可以跑全部）
├── model.py               ← 模型定義（SIR 函數等）
├── optimization.py        ← 最佳化相關函數
├── visualization.py       ← 畫圖相關函數（若有）
├── requirements.txt       ← 所需套件列表
└── outputs/               ← 輸出的圖表（.png 檔）
    ├── sir_model.png
    ├── convergence_test.png
    └── sensitivity_analysis.png
```

> 若你的專題很小，可以只有一個 `main.py` + `README.md`，不需要拆很多檔案。

#### 要求 2：每個函數要有 docstring（中文說明）

```python
def peak_infected(v, beta=0.3, gamma=0.1, N=10000, I0=10):
    """
    計算給定接種比例 v 時的感染高峰人數。
    
    這個函數作為最佳化的目標函數，輸入接種比例 v，
    透過數值求解 SIR 微分方程組，回傳感染高峰人數。
    
    參數：
        v     (float) : 接種比例，範圍 [0, 1]
        beta  (float) : 接觸率，預設值 0.3
        gamma (float) : 康復率，預設值 0.1
        N     (int)   : 總人口，預設值 10000
        I0    (int)   : 初始感染者數，預設值 10
    
    回傳：
        float : 感染高峰人數
    
    範例：
        >>> peak = peak_infected(0.67)
        >>> print(f"67% 接種率下的感染高峰：{peak:.0f} 人")
    """
    # ... 實作 ...
```

#### 要求 3：主程式的關鍵段落要有中文注釋

```python
# ================================================================
# 步驟一：設定模型參數
# 參數來源：World Health Organization (2020) 對 COVID-19 的估計
# beta = 0.3（每天每個感染者平均接觸 0.3 個易感者）
# gamma = 0.1（平均感染時間約 10 天）
# ================================================================
```

#### 要求 4：README.md 的基本結構

```markdown
# [你的專題名稱]

## 專題簡介
一段話說明這個專題在做什麼、研究什麼問題。

## 研究問題
在 [條件] 下，[什麼] 是最佳的 [決策變數]？

## 方法
- 模型：SIR 傳染病模型（3 個微分方程）
- 數值方法：`scipy.integrate.solve_ivp`（RK45）
- 最佳化：`scipy.optimize.minimize_scalar`（bounded）

## 主要結果
在基準條件（β=0.3, γ=0.1, N=10000）下，
最佳接種比例為 **67%**，對應感染高峰 **285 人**（佔總人口 2.85%）。

## 如何執行
1. 安裝依賴：`pip install -r requirements.txt`
2. 執行主程式：`python main.py`
3. 輸出圖表會存在 `outputs/` 資料夾

## 環境需求
- Python 3.8+
- numpy >= 1.20
- scipy >= 1.7
- matplotlib >= 3.4

## 技術限制
- 模型假設均勻混合，忽略地理聚集效應
- 參數為固定值，未考慮疫情期間介入措施的變化
- 使用局部最佳化方法，若目標函數有多個極值需額外驗證
```

### 未來延伸方向

整理作品集時，思考這個專題可以往哪裡走：

**技術升級方向**：
- 加入隨機性（SEIR 隨機模型）
- 多組群模型（年齡分層的 SIR）
- 加入控制理論（動態調整接種策略）

**競賽方向**：
- 科展：台灣中小學科學展覽會、全國科學探究競賽
- 程式競賽：若有 optimization 組的競賽

**申請用途**：
- 大學申請備審：證明有自主研究能力
- 申請大學研究室：作為聯繫時的作品展示
