# Level 2 週 01：課綱與範例專題拆解

## 本週學習目標

1. 學生了解 Level 2 的完整課程流程與期末成果形式
2. 學生能分辨 AP Research / IB IA / A Level 的評分重點（探究、模型、分析、反思）
3. 學生從 1–2 個示範範例中，理解「好的模擬型研究」長什麼樣子
4. 學生初步發散出 2–3 個自己有興趣的題目方向

---

## 課堂時間分配（共 120 分鐘）

| 段落 | 時間 | 內容 |
|------|------|------|
| 暖身回顧 | 10 分鐘 | 介紹本課程與 Level 1 的差異，互相認識（若為新班） |
| 概念講解 | 25 分鐘 | AP/IB/A-Level 評分架構介紹 |
| 範例示範 | 30 分鐘 | 拆解 2 個範例專題的結構 |
| 學生實作 | 35 分鐘 | 發散題目方向，填寫興趣探索表 |
| 回顧與預告 | 20 分鐘 | 總結本週重點，說明作業，預告週 02 |

## 各段落執行細節

### 暖身回顧（10 分鐘）

**若為全新學生（未從 Level 1 直升）：**
- 簡短自我介紹（姓名、年級、對哪類問題有興趣）
- 老師說明：Level 2 不是從頭學 Python，而是從「有基礎」出發做研究

**若為 Level 1 直升：**
- 提問：「Level 1 你做的專題是什麼？如果可以繼續做，你想往哪個方向深入？」

**共同引入問題：**
> 「如果有人問你，你的 Python 專題研究了什麼、發現了什麼——你能在 3 分鐘內說清楚嗎？」
> 這正是 AP/IB 評分者在問的問題，也是這 10 週課程要讓你做到的事。

---

### 概念講解（25 分鐘）

**重點：AP / IB IA / A Level 研究型作業的共同評分架構**

不需要逐一介紹各考試制度，聚焦在**共通邏輯**：

1. **探究（Research Question + Background）**：問題有沒有意義？有沒有聚焦？
2. **方法（Model + Simulation Design）**：怎麼用程式去回答問題？假設合不合理？
3. **分析（Results + Interpretation）**：數字和圖說了什麼？有沒有真正分析，而不是只貼圖？
4. **反思（Discussion + Limitations）**：結果回答了問題嗎？模型有哪些限制？

**板書 / 投影片結構：**
```
研究問題（RQ）
    ↓
模型設計（假設 + 參數）
    ↓
模擬實驗（跑程式、改參數）
    ↓
結果分析（圖表 + 解讀）
    ↓
討論與限制
```

**關鍵提問（確認理解）：**
- 「如果結果很好看但模型假設不合理，會扣分嗎？」（會，方法分會低）
- 「如果 Python 跑出漂亮的圖但沒有文字解釋，可以嗎？」（不行，分析分會低）

---

### 範例示範（30 分鐘）

**不 live coding，改為：老師帶著學生「讀範例」**

**示範範例 A：排隊系統（較簡單，接近 Level 1）**
拆解要點：
- RQ 是什麼？（增加一個收銀台，等待時間減少多少？）
- 模型假設了什麼？（顧客到達是 Poisson、服務時間是常數）
- 圖表說了什麼？（橫軸 = 顧客到達率，縱軸 = 平均等待時間）
- 討論寫了什麼？（結論連回 RQ，指出模型不考慮急件插隊的限制）

**示範範例 B：疾病傳播 SIR 模型（稍難）**
拆解要點：
- RQ 是什麼？（接觸率如何影響感染高峰出現的時間？）
- 模型裡有哪些「角色」？（S / I / R 三個狀態）
- 圖表說了什麼？（不同 β 值下，I 的高峰時間點不同）
- 限制是什麼？（假設人口均質混合，沒考慮空間分布）

**引導問題（讓學生主動發現）：**
- 「這個範例中，最核心的假設是什麼？」
- 「如果你要改進這個模型，你會加什麼？」

### 學生實作（35 分鐘）

**任務：填寫「題目探索草稿」**

發給學生（或投影以下問題）：

```
題目方向發散（寫 2–3 個，不需要全想清楚）

題目 1：我對 [___] 這個現象有興趣，我想探討 [___] 怎麼影響 [___]
題目 2：（同上格式）
題目 3：（同上格式）

初步檢查（對每個題目）：
- [ ] 可以用「模擬」的方式研究嗎？
- [ ] 有一個可以量化的輸出嗎？
- [ ] 題目不會太大（不是「模擬整個經濟體」）？
```

老師巡視時的提示方向：
- 若學生想不出題目：問「你喜歡玩什麼遊戲、看什麼科目、對什麼事情好奇？」
- 若題目太大：引導問「如果只做一個最簡單的版本，會是什麼？」
- 若題目太小（例如「骰子」）：說「這是 Level 1 等級，可以加入時間、空間或多個變數嗎？」

### 回顧與預告（20 分鐘）

**本週 3 個關鍵點（板書）：**
1. Level 2 的終點是：一份完整報告 + 可展示的簡報，而不只是一個程式
2. 評分者在乎：RQ 清不清楚、模型假設合不合理、分析有沒有連回問題
3. 題目來自你自己的興趣——這週只需要有方向，不需要決定

**作業說明：** 見 `level-2-week-01-assignment-brief.md`

**預告週 02：**
> 「下週我們會學一個標準的模型拆解方式，然後老師示範一個完整的模擬案例——從問題到圖表。你可以把你的題目帶來，我們會試著一起把它畫成模型草圖。」

## 常見問題與處理方式

| 常見問題 | 建議回應 |
|---------|---------|
| 「我不知道要做什麼題目」 | 「沒關係，這週不需要決定，只需要列出你對什麼有好奇心。下週我們一起篩選。」 |
| 「我的題目太簡單了吧？」 | 「先說說你的題目，我們一起看看能怎麼讓它更有深度。」 |
| 「AP/IB 跟我沒關係，我為什麼要了解那個格式？」 | 「這個格式其實就是『說清楚一個研究』的基本邏輯，無論是競賽、申請還是給老師看，都適用。」 |
| 「Level 1 沒做過的學生可以直接接上嗎？」 | 「可以，只要你可以用 Python 寫迴圈和函數，這週的概念和你有關。」 |

## 教師備用材料

### 排隊系統示範範例架構（簡版）

```python
import random
import matplotlib.pyplot as plt

def simulate_queue(arrival_rate, service_time, n_customers=200):
    """模擬單服務台排隊系統，回傳平均等待時間"""
    wait_times = []
    queue_end = 0  # 目前服務台何時空閒
    for i in range(n_customers):
        arrival = i / arrival_rate + random.expovariate(arrival_rate)
        start_service = max(arrival, queue_end)
        wait_times.append(start_service - arrival)
        queue_end = start_service + service_time
    return sum(wait_times) / len(wait_times)

rates = [0.5, 1.0, 1.5, 2.0, 2.5]
avg_waits = [simulate_queue(r, 1.0) for r in rates]

plt.plot(rates, avg_waits, marker='o')
plt.xlabel("到達率（顧客/分鐘）")
plt.ylabel("平均等待時間（分鐘）")
plt.title("到達率 vs. 平均等待時間")
plt.grid(True)
plt.show()
```

> 上課時不需要打出這段 code，用於老師自己準備範例時參考。示範時只拆解結構。
