# Level 3 週 09：技術寫作與簡報

## 本週主題

這週從「寫程式」切換到「寫文字」。你的程式已經跑出結果了——現在要用語言說清楚你做了什麼、為什麼這樣做、以及你的方法有什麼限制。

本週兩個任務：
1. **技術寫作**：完成報告的「方法」段和「實驗設計」段草稿
2. **簡報準備**：了解 Level 3 進階簡報的結構，開始準備期末展示

---

## 核心概念一：方法章的結構

### 方法章回答四個問題

| 問題 | 範例 |
|------|------|
| 我用什麼模型？ | 「本研究採用 SIR 傳染病模型，以三個微分方程描述易感者（S）、感染者（I）和康復者（R）的時間演化。」 |
| 我用什麼工具？ | 「數值求解採用 Python SciPy 套件的 `solve_ivp` 函數（RK45 自適應步長方法）。」 |
| 我怎麼設定關鍵參數？ | 「最大積分步長設為 0.5 天，經收斂測試確認誤差小於 0.1%；最佳化採用 `minimize_scalar` 的 bounded 方法，搜尋範圍為 [0, 0.99]。」 |
| 我為什麼這樣選？ | 「RK45 方法相較於固定步長歐拉法，能自動調整步長以在精度和計算效率之間取得平衡；minimize_scalar 對一維搜尋問題的求解比通用 minimize 更精確。」 |

## 核心概念二：從程式碼翻譯到方法章文字

### 翻譯練習

以下是程式碼片段，比較兩種翻譯方式：

**原始程式碼**：
```python
result = minimize_scalar(
    peak_infected,
    bounds=(0.0, 0.99),
    method='bounded'
)
print(result.x, result.fun)
```

**不好的翻譯（直接照翻）**：
「我用 `minimize_scalar` 函數，設定 `bounds=(0.0, 0.99)`，`method='bounded'`，然後 print 出結果。」

**好的翻譯（說明原理和理由）**：
「本研究以 SciPy 的 `minimize_scalar` 函數（bounded 方法）搜尋使感染高峰最小的接種比例，搜尋範圍限制在 [0%, 99%] 之間（0 代表不接種，99% 為實際可行的上限）。bounded 方法在有界搜尋問題上比無界方法更穩定，適合接種比例這類有明確範圍的連續變數。」

**兩者的差別**：
- 好的翻譯說了「為什麼是 0–0.99」（不是 0–1，因為現實中 100% 接種不可行）
- 好的翻譯說了「為什麼選 bounded 方法」（比無界方法穩定）
- 好的翻譯沒有出現 `print` 這種程式操作

## 核心概念三：技術貢獻與技術限制

### 技術貢獻：比最簡單的版本多做了什麼

你的技術貢獻 = 「我做了 X，而 X 比直接用 for 迴圈或手動調參好，因為…」

**範例**：
- 「本研究採用自適應步長 ODE solver，相較於固定步長差分方法，在相同計算時間下精度更高（收斂測試顯示誤差減少約 10 倍）。」
- 「透過數值最佳化，本研究自動找到使感染高峰最小的接種比例，避免了手動網格搜尋的低效與不精確。」

### 技術限制：你的假設在哪裡可能失效

**三個標準框架**，任何專題都可以用：

**框架 1：模型假設的限制**
「本模型假設 [A]，若 [A] 不成立（如 [反例]），模型結果可能 [偏差方向]。」

範例：「本模型假設所有個體的接觸率相同（均勻混合），忽略了家庭、學校等社交結構的影響。在實際疫情中，高度聚集的社交環境（如學校或工廠）會導致局部傳播遠高於平均值，因此模型可能低估局部爆發的嚴重性。」

**框架 2：數值方法的限制**
「本方法在 [特定條件] 下 [可能失效/精度下降]，例如 [具體情境]。」

範例：「當接觸率 β 極大時（β > 2），ODE 系統變為 stiff（各狀態量的時間尺度差異極大），RK45 方法可能需要極小的步長才能收斂，此時應改用 `method='Radau'` 等隱式積分法。本研究的參數範圍（β ≤ 0.6）未進入此區域，但對於更高傳播率的病毒，需要額外驗證。」

**框架 3：參數不確定性的限制**
「模型參數 [P] 來自 [來源/估計方式]，若真實值與 [P] 不同，最佳解可能 [如何改變]（敏感度分析結果見圖 X）。」

## 寫作練習

### 練習 1：翻譯你的程式碼

找出你的專題程式中最重要的一段（通常是呼叫 SciPy 的那段），用「好的翻譯」方式寫成方法章文字。

**程式碼（抄一段在這裡）**：
```python
# 你的程式碼
```

**你的翻譯草稿（150 字以上）**：

（在這裡寫）

---

### 練習 2：寫技術限制

根據你的專題，用上面三個框架各寫一個限制句。

**框架 1（模型假設）**：
「本模型假設 ___，若 ___ 不成立（如 ___），模型結果可能 ___。」

**框架 2（數值方法）**：
「本方法在 ___ 條件下 ___，例如 ___。」

**框架 3（參數不確定性）**：
「模型參數 ___ 來自 ___，若真實值不同，最佳解可能 ___（見敏感度分析圖）。」

## 進階簡報結構指南

### Level 3 簡報 vs Level 2 簡報

Level 3 的期末展示是技術性展示，受眾假設為「有數學/科學背景的人」。

**建議投影片結構（10–12 張，約 6 分鐘）**：

| 投影片 | 標題 | 主要內容 |
|--------|------|----------|
| 1 | 問題與動機 | 你要解決什麼問題？為什麼值得研究？ |
| 2 | 研究問題（RQ） | 具體的問題陳述（一句話） |
| 3–4 | 模型與假設 | 核心方程式、狀態變數定義、假設列表 |
| 5 | 方法 | 使用的工具（具體函數名）、關鍵參數 |
| 6 | 收斂測試 | 展示你確認了數值精度（一張圖） |
| 7–8 | 主要結果 | 模型輸出圖、最佳化結果 |
| 9 | 敏感度分析 | 一個參數的變化如何影響最佳解 |
| 10 | 限制與未來方向 | 至少 2 個技術限制；如果繼續做可以做什麼 |
| 11（選） | 程式架構圖 | 用方塊圖說明程式各部分的關係 |

---

### 程式架構圖範例

用文字方塊圖說明你的程式架構（可以用白板、Figma 或手繪）：

```
[輸入參數]
  β, γ, N, I0, v
       ↓
[SIR ODE 模型]
  sir_model(t, y, β, γ, N)
       ↓
[數值求解]
  solve_ivp(RK45, max_step=0.5)
       ↓
[目標函數計算]
  peak_infected(v) → 單一數字
       ↓
[最佳化]
  minimize_scalar(bounded, [0, 0.99])
       ↓
[敏感度分析]
  改變 β，重複最佳化 × 10
       ↓
[結果輸出]
  圖表 + 最佳解數值
```

### 展示時的技術問答準備

Level 3 的展示後，老師（或觀眾）可能問的技術問題：

1. 「你為什麼選 RK45 而不是歐拉法？」
   → 準備答案：「RK45 是自適應步長，在精度相同的情況下，比固定步長歐拉法計算更有效率。」

2. 「你的最佳解是局部解還是全局解？你怎麼知道？」
   → 準備答案：「我試了 3–5 個不同初始點，結果一致，說明很可能是全局解。但不能 100% 保證。」

3. 「你的模型最不可靠的假設是什麼？」
   → 準備答案：用你在技術限制中寫的內容

4. 「敏感度分析中，哪個參數的影響最大？」
   → 準備答案：看你的圖，哪條曲線斜率最大
