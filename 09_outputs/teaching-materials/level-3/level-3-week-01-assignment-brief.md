# Level 3 週 01 作業單
## 作業名稱：我的舊專題升級說明

---

## 作業目標

讓你在進入 NumPy / SciPy 技術學習之前，先清楚地想清楚「自己的升級方向」。這份作業是你整個 Level 3 的起點，後續幾週的實作都會基於這裡的計畫來進行。

---

## 繳交清單

| # | 項目 | 格式 | 必/選 |
|---|------|------|-------|
| 1 | 舊專題程式碼（或連結） | `.py` / `.ipynb` / GitHub URL | 必交 |
| 2 | 升級說明文件 | `.md` 或 `.txt`，至少 200 字 | 必交 |
| 3 | 升級計畫草圖（選填） | 手繪照片或 `.png` 流程圖 | 選交 |

**繳交方式**：上傳至課程 Google Drive 資料夾，或以 Email 傳給老師  
**繳交期限**：下次上課（Week 2）前 24 小時

---

## 具體要求

### 要求 1：繳出可以執行的舊專題程式碼

- 如果你在 Level 2 有完整的專題程式，把它整理好（有基本的中文或英文注釋）並繳交
- 如果程式分散成多個檔案，整理成一個主程式 `main.py` 或一個 Jupyter Notebook
- 程式必須在你的電腦上可以正常執行（不要繳交有 bug 的版本）

### 要求 2：升級說明文件需包含以下四段

**第一段：舊專題簡介（約 50 字）**  
說明你的問題是什麼、你用什麼方法解決、程式的主要功能。

**第二段：技術現況（約 50 字）**  
說明目前程式的技術特徵：用了什麼資料結構？有沒有 for 迴圈？計算量大嗎？有哪些限制？

**第三段：升級目標（約 50–100 字）**  
說明你打算在 Level 3 做哪些技術升級。至少提到一個具體工具（NumPy / SciPy optimize / SciPy odeint / SciPy integrate），並說明為什麼選它。

**第四段：預期成果（約 50 字）**  
升級後，你的程式、報告或研究問題會有什麼具體改善？

### 要求 3：升級目標要和 Level 3 課程工具對應

你的升級說明中，至少要提到以下其中一種：
- 用 NumPy 改善計算效率（對應 Week 2）
- 用 SciPy 做求根或積分（對應 Week 3）
- 用 SciPy odeint 模擬動態系統（對應 Week 4）
- 用 SciPy minimize 找最佳解（對應 Week 5）

### 要求 4：說明要具體，不要空泛

**不好的例子**：「我想讓程式更好更快更有效率。」  
**好的例子**：「我的族群模擬用了 10 個巢狀 for 迴圈，跑 1000 次模擬要 8 秒。我計畫用 NumPy 向量化，目標是把時間壓到 0.5 秒以下，並加上 `np.random.normal` 做參數的不確定性分析。」

### 要求 5：用正確的工具名稱與函數名稱

在說明中提到工具時，請使用正確的名稱：
- 寫 `numpy`（不是「numpy 函式庫」、「numpy 套件」等）
- 寫 `scipy.optimize.minimize`（而不只是「scipy」）
- 這個習慣從現在開始建立，報告寫作時很重要

## 自我檢查清單

在繳交前，請確認以下每一點：

- [ ] 我的舊專題程式碼可以在我的電腦上執行
- [ ] 程式碼有基本注釋（每個函數或主要段落有一行說明）
- [ ] 升級說明文件有四個段落（簡介、現況、目標、預期成果）
- [ ] 升級目標明確提到至少一個 Level 3 工具，並說明為什麼選它
- [ ] 說明夠具體：有提到具體的計算步驟、數量級、函數名稱等
- [ ] 文件字數至少 200 字
- [ ] 檔案已上傳到正確的資料夾，並確認老師可以開啟
