# 入班評估、升級與轉班政策

> 本文件為**行政操作層**規範，完整的升降級邏輯請參照 `01_curriculum/progression-rules.md`。

---

## 報名時的入班評估流程

### 標準入班（Level 1）

無需程度測驗，具備以下基本條件即可報名：

- 有 Python 基礎語法概念（變數、迴圈、函數）
- 能在自己電腦安裝 Python 並執行程式

> 若學生完全零基礎（從未寫過 Python），需先完成補習班現有 Python 入門課程。

### 直接入班高級 Level（跳過 Level 1 或更多）

1. 報名時勾選「直接入班申請」
2. 老師安排 **30 分鐘線上程度測驗**（測驗前先填寫基本背景表）
3. 測驗結束後 **3 個工作天內**回覆結果
4. 通過 → 直接報名目標 Level
5. 未通過 → 建議降一個 Level 報名，說明原因

測驗分流標準詳見 `01_curriculum/progression-rules.md`。

## 課程內升班規則

### 標準升班（完成本 Level → 下一 Level）

- 每 Level 課程結束後，**預設可直接升班**，無需再次測驗
- 老師在**期末 1-on-1**中確認學生意願與準備狀況
- 建議升班時間：下一梯次開始前完成確認

### 升班流程

1. 老師在第 10 週 1-on-1 提出升班建議（或學生主動詢問）
2. 確認學生意願、下一梯次時間
3. 通知家長（升班建議 + 費用說明）
4. 學生/家長完成報名手續（含繳費）

> **升班優惠**：直接升下一 Level 的在讀學生享有 **9 折續班價**（詳見 `04_business/discounts.md`）

## 降級規則

### 觸發條件

以下任一情況，老師需主動評估是否建議降級：

1. 連續 2 週作業 rubric 中 ≥ 3 項為「待改」
2. 第 5 週中期分享時，程式無法執行或完全無產出
3. 學生明確表示壓力過大、跟不上進度

### 降級流程

1. 老師在 **1-on-1** 私下說明觀察（不當眾提出）
2. 說明降級理由與效益（「先做紮實，之後進步更快」）
3. 通知家長（書面說明：原因、建議降至哪一 Level、費用差額）
4. 確認學生與家長同意後，辦理轉班（退費差額退回，或抵扣下期費用）

> 降級為例外處理，不應在第 3 週以前建議（需有足夠觀察資料）。

## 同 Level 重修規則

**適用情況**：

- 完成某 Level 但期末成果明顯不完整，想補強後再升
- 學生本人希望把這一級的專題做得更完整

**規定**：

- 重修按當期定價計費（無特殊折扣）
- 1-on-1 聚焦於上次未完成的部分
- 重修不影響之後升班資格

## 轉班規則

### 同 Level 不同梯次轉班

| 時間點 | 規則 |
|--------|------|
| 開課前 **7 天以上** | 免費轉至下一梯次（需有名額） |
| 開課前 **7 天以內** | 視為退費後重新報名（依退費政策） |
| 課程開始後 | 視為退費後重新報名（已上週次不退費） |

### 跨 Level 轉班（升班 / 降班）

- 依上述升降級流程重新評估
- 費用差額：多退少補

---

## 入班測驗管理

- 測驗以線上形式進行（Google Form 或 Jupyter Notebook 作答）
- 測驗結果由老師保存，作為分流依據
- 測驗結果不對外公開（僅老師與學生/家長共享）
- 測驗資料保存至該學生完成本課程為止
