# Level 3 週 03 作業單
## 作業名稱：SciPy 數值求根或積分小範例

---

## 作業目標

完成一個獨立的小程式，使用 `scipy.optimize.brentq` **或** `scipy.integrate.quad`（或兩者都用），解決一個有實際意義的問題。重點不只是「讓程式跑通」，而是要能說明：「這個計算在解決什麼問題？結果的意義是什麼？」

---

## 繳交清單

| # | 項目 | 格式 | 必/選 |
|---|------|------|-------|
| 1 | 主程式 | `.py` 或 `.ipynb` | 必交 |
| 2 | 圖表輸出（至少一張） | 圖內嵌在 notebook，或另存 `.png` | 必交 |
| 3 | 結果說明（見要求 3） | 程式碼注釋或 Markdown 格子 | 必交 |

**繳交方式**：上傳至課程 Google Drive  
**繳交期限**：Week 4 上課前 24 小時

---

## 具體要求

### 要求 1：選擇一個有實際意義的問題

**不接受**的題目：「計算 x^2 的積分」（沒有實際意義，可以用公式算）

**可接受**的題目類型：

| 分類 | 題目例子 |
|------|----------|
| 物理 / 工程 | 找彈簧在某個力下達到平衡的位置；計算物體在變速度下的位移 |
| 生物 / 生態 | 找族群達到攜帶容量一半的時間；計算某段時間內的族群增長總量 |
| 經濟 / 商業 | 找供需均衡價格；計算某定價策略的總收益 |
| 機率 / 統計 | 計算某分布在一段區間內的機率；找中位數（讓 CDF = 0.5 的點） |
| 你的舊專題 | 把你的舊專題中某個計算改用 SciPy 方法 |

### 要求 2：程式必須包含這四個步驟

**步驟一：定義問題（用注釋說明）**
```python
# 問題：[說明你要解決什麼問題]
# 數學形式：[寫出你用的方程式或積分式]
```

**步驟二：畫圖（求根時必須；積分時強烈建議）**

求根時：畫出函數圖，在圖上標記你找到的根  
積分時：畫出函數圖，用 `fill_between` 標記積分區域

**步驟三：呼叫 SciPy 函數並儲存結果**
```python
# 求根
root = brentq(f, a, b)

# 積分
result, error = quad(f, a, b)
```

**步驟四：驗證或說明結果**

如果有解析解：計算解析解並比較  
如果沒有解析解：說明結果的物理或問題意義，確認結果在「合理範圍」內

### 要求 3：在程式碼末尾加上 2–3 句結果說明

```python
# 結果說明：
# 1. [數值結果是什麼？]
# 2. [這個結果在物理/問題意義上代表什麼？]
# 3. [誤差是多少？你認為這個精確度足夠嗎？]
```

例如：
```python
# 結果說明：
# 1. 求根結果：均衡價格 p* = 36.67
# 2. 這代表當價格為 36.67 時，供給量等於需求量（均為 63.33 單位）
# 3. brentq 精確到機器精度，對這個問題完全足夠
```

### 要求 4：程式必須完整可執行

包含所有 `import`，從頭到尾執行不出錯。

### 要求 5（挑戰選項）：結合求根和積分

如果你有餘力，嘗試在同一個程式中同時用到求根和積分。

例如：
- 先用求根找到某個臨界點
- 再用積分計算在那個臨界點之前的累積量
