# Level 3 週 06 作業單：高級專題計畫書

## 作業名稱與目標

**作業名稱**：高級專題計畫書（Advanced Project Plan）

**目標**：
在中期展示的口頭說明基礎上，寫出完整的書面方法計畫，讓老師確認你的題目、工具選擇和實驗設計合理，可以進入下週的實作階段。

---

## 繳交清單

| 項目 | 格式 | 說明 |
|------|------|------|
| 高級專題計畫書 | `.md` 或 Google Doc | 見下方格式要求 |
| 程式骨架（選繳） | `.py` 檔 | 若已開始寫可一起繳，不做強制要求 |

**截止時間**：下次 group 課前（週 07 上課前 24 小時）

---

## 格式要求：高級專題計畫書

請依照以下五個段落撰寫，每段 3–8 行。全文不必超過一頁 A4（約 500 字）。

### 段落 1：問題陳述
- 描述你要研究的問題
- 說明為什麼這個問題有趣或有意義
- 說明你的假設（哪些因素你會忽略不計）

### 段落 2：模型說明
- 描述你的數學模型（可以用文字 + 方程式）
- 說明模型的輸入變數和輸出指標

### 段落 3：擬用工具
- 具體寫出你計畫使用的 Python 函數（例如：`scipy.integrate.solve_ivp`、`scipy.optimize.minimize`）
- 說明為什麼選這個工具（而不是自己用 for 迴圈）

### 段落 4：實驗設計
- 說明你打算改變哪些參數（控制變數）
- 說明你要觀察什麼結果指標（輸出變數）
- 說明你預計產出什麼圖表或數字

### 段落 5：預計困難
- 說明你預期在實作中最難處理的技術問題
- 說明你打算如何應對（例如：先做簡化版本、查 SciPy 文件）

---

## 具體要求

1. **工具名稱必須具體**：不能只寫「用 SciPy」，要寫到函數名（如 `scipy.integrate.solve_ivp`）
2. **實驗設計必須有對照**：要明確說「固定 X，改變 Y，觀察 Z」的結構
3. **假設要明說**：任何你的模型沒有考慮到的因素，請在問題陳述中明確列出
4. **問題要夠小**：能在 3 週內完成的問題才算合格（不是 3 個月的大型研究）
5. **符合 Level 3 技術深度**：若整個計畫用 for 迴圈就能完成，請重新設計

---

## 自我檢查清單

繳交前，請確認以下每一項：

- [ ] 問題陳述中有明確的假設條件
- [ ] 模型說明中有具體的方程式或計算邏輯（不只是文字描述）
- [ ] 擬用工具中寫出了具體的 Python 函數名
- [ ] 實驗設計符合「固定 X，改變 Y，觀察 Z」的結構
- [ ] 預計困難不是「我不知道會有什麼困難」（至少寫出一個具體的技術問題）
- [ ] 全文不超過一頁、段落結構清楚、沒有錯字

---

## 常見錯誤提示

**太模糊的問題**：「我要模擬一個系統。」
→ 改為：「我要模擬一個雙槽水位系統，在不同進水流量下，分析兩個水槽何時達到穩態。」

**工具不具體**：「我會用 Python 的科學計算工具。」
→ 改為：「我使用 `scipy.integrate.solve_ivp` 搭配 `method='RK45'`，時間範圍設為 0–100 秒。」

**沒有對照的實驗**：「我要跑模擬然後看結果。」
→ 改為：「我固定進水流量為 2L/s，改變管道截面積（0.01 到 0.05 m²，共 5 個值），觀察達到穩態的時間。」
