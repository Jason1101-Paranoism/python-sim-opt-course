# Level 3 週 09：技術寫作與簡報（教師教案）

## 本週學習目標

1. 學生能寫出「方法」章節，清楚說明使用的數值方法、工具名稱、關鍵參數設定，以及為什麼這樣選擇
2. 學生能寫出「實驗設計」章節，清楚說明各組實驗的條件與預期比較方向
3. 學生能說明自己的技術貢獻（「我做了什麼」）和技術限制（「這個方法在什麼條件下失效」）
4. 學生能準備一份偏技術細節的進階簡報（包含程式架構圖和結果分析）

---

## 課堂時間分配（正常週結構，120 分鐘）

| 段落 | 時間 | 具體內容 |
|------|------|----------|
| 暖身回顧 | 10 分鐘 | 確認最佳化作業完成狀況；每人說「我的最佳解是什麼、意義是什麼」（一句話） |
| 概念講解 | 25 分鐘 | 方法章的結構；技術貢獻 vs 技術限制的寫法；進階簡報的設計原則 |
| 範例示範（Live coding / 寫作示範） | 30 分鐘 | 示範如何把程式碼翻譯成方法章文字；示範好的 vs 差的寫作例子 |
| 學生實作 | 35 分鐘 | 學生動筆寫方法段和實驗設計段草稿 |
| 回顧與預告 | 20 分鐘 | 總結三個關鍵點；說明作業；預告下週期末展示流程 |

---

## 各段落執行細節

### 暖身回顧（10 分鐘）

**提問序列**：
1. 「上週的最佳化跑出來了嗎？最佳解是多少？」（快速點名，每人說一句）
2. 「有沒有人的最佳化結果讓你感到意外？為什麼？」（找 1–2 人分享）
3. 「今天我們要把你做的這些，用文字說清楚。說清楚比做出來更難——但你說得清楚，才代表你真的懂。」

---

### 概念講解（25 分鐘）

#### 一、方法章的結構與功能（10 分鐘）

**方法章回答的問題**（白板寫出）：
1. 我用什麼模型？（方程式或架構描述）
2. 我用什麼工具？（具體函數名稱）
3. 我怎麼設定關鍵參數？（初始條件、步長、收斂條件）
4. 我為什麼這樣選？（說理由，不只說事實）

**方法章不是什麼**：
- 不是程式碼的翻譯（把 Python 照樣翻成中文，沒有意義）
- 不是操作手冊（「先打開 Python，然後 import numpy」）
- 不是結果描述（那是結果章的事）

**受眾設定**：「想像你的讀者懂高中數學，知道什麼是微分，但沒看過你的程式。」

---

#### 二、技術貢獻 vs 技術限制（10 分鐘）

**技術貢獻**：你做了什麼別人沒做到的事（相對於最簡單的基準版本）

範例對比：

| 不夠好的說法 | 好的說法 |
|-------------|----------|
| 「我用了 SciPy 求解 ODE」 | 「本研究採用 RK45 自適應步長方法（`solve_ivp`），相較於固定步長的歐拉法，在保持同等精度下計算效率提升約 3 倍（以收斂測試驗證）」 |
| 「我做了最佳化」 | 「本研究使用 `minimize_scalar` 的 bounded 方法，在接種比例 [0, 0.99] 的可行域內搜尋使感染高峰最小的接種策略」 |

**技術限制**：你的方法在什麼條件下可能失效

**三個標準限制框架**（任何專題都可用）：
1. **模型假設的限制**：「本模型假設 [X]，若 [X] 不成立，結果可能偏差」
2. **數值方法的限制**：「本方法在 [某種極端條件] 下可能不收斂 / 精度下降」
3. **資料或參數的限制**：「β 和 γ 的值是根據 [來源] 估計，若真實值不同，最佳解也會不同」

---

#### 三、進階簡報的設計（5 分鐘）

Level 3 的簡報與 Level 2 的差異：

| 面向 | Level 2 | Level 3 |
|------|---------|---------|
| 聽眾假設 | 一般人 | 有技術背景的人 |
| 程式碼 | 不出現 | 可以出現（關鍵片段） |
| 圖表 | 結果圖 | 結果圖 + 收斂測試圖 + 敏感度分析圖 |
| 說明深度 | 「做了 X，得到 Y」 | 「為什麼選 X，X 有什麼限制，Y 背後的原因是 Z」 |

**進階簡報的必要投影片**：
1. 問題與動機（1 張）
2. 模型與假設（1–2 張，含方程式）
3. 方法（1–2 張，含工具說明）
4. 結果（2–3 張）
5. 敏感度分析（1 張）
6. 限制與未來方向（1 張）

---

### 範例示範（30 分鐘）— 寫作示範

**本週示範形式**：不是 live coding，而是在文字編輯器中**示範寫作**（邊寫邊說明）

**從程式到方法章的翻譯示範**：

老師先顯示程式碼片段：

```python
sol = solve_ivp(
    sir_model, (0, 160), y0,
    args=(0.3, 0.1, 10000),
    dense_output=True,
    max_step=0.5
)
```

然後示範翻譯過程（逐步建立文字）：

**第一版（直譯，不好）**：
「我用 `solve_ivp` 函數，參數設定為 `t_span=(0, 160)`，`max_step=0.5`，`dense_output=True`。」

→ 這只是把程式碼用中文寫一遍，沒有說明為什麼。

**第二版（加入理由，好）**：
「本研究採用 SciPy 的 `solve_ivp` 函數（Runge-Kutta 4-5 階方法，RK45），以數值方式求解 SIR 微分方程組。模擬時間範圍設為 0 至 160 天（足以涵蓋疫情從爆發到穩定的完整過程），最大積分步長設為 0.5 天（經收斂測試確認在此步長下，感染高峰估計值的變化小於 0.1%）。」

老師說明差異：
- 第二版說了「為什麼是 160 天」（疫情的自然周期）
- 第二版說了「為什麼 max_step=0.5」（收斂測試的結果）
- 第二版說了「RK45 是什麼」（讀者不一定知道 solve_ivp 背後的算法）

**再示範一個限制段落**：

「本模型的主要限制在於：（1）均勻混合假設——模型假設每個人與其他人接觸的機率相等，忽略了人口的地理分布和社交結構；（2）固定參數——接觸率 β 和康復率 γ 在整個模擬期間視為常數，但實際疫情中這些參數可能隨時間或介入措施而改變；（3）局部最佳化——使用 `minimize_scalar` 的 bounded 方法，僅保證局部最佳解，對於可能存在多個局部最小值的情況需要進一步驗證。」

---

### 學生實作（35 分鐘）

**任務說明**：
「現在打開一個文字文件，開始寫你的方法段和實驗設計段。目標：寫完一個完整段落（至少 100 字）。」

**教師巡視重點**：
- 學生是否在寫「為什麼」，而不只是「做了什麼」
- 是否有工具名稱（具體到函數名）
- 是否有至少一個技術限制

**卡關學生的引導**：
- 「你的程式裡最重要的函數是哪個？先說那個函數做了什麼事。」
- 「你的讀者可能會問哪個問題？先回答那個問題。」
- 「用三句話說你的方法：用了什麼、為什麼用這個、有什麼限制。」

---

### 回顧與預告（20 分鐘）

**本週三個關鍵點**（白板寫下）：
1. 方法章要說「為什麼」，不只說「做了什麼」
2. 技術貢獻 = 你做了什麼超過最簡單基準版本的事；技術限制 = 你的假設在哪裡可能失效
3. 進階簡報的受眾假設比 Level 2 更有技術背景，可以展示收斂測試圖和方程式

**作業說明**：
- 完成方法段草稿（150–250 字）+ 實驗設計段草稿（100–200 字）
- 開始準備期末展示簡報（下週直接展示，不再有大幅修改時間）

**預告下週**：「下週是期末展示。每人 5–8 分鐘，展示你從第 6 週以來做的完整高級專題。這是一個正式的技術展示，不是隨便說說。」

## 常見問題與處理方式

| 問題 | 處理方式 |
|------|----------|
| 「我不知道方法章要寫多長」 | 給參考：150–300 字，要能說清楚「用了什麼工具、怎麼設定、為什麼這樣選」 |
| 「我的方法很簡單，沒什麼好寫的」 | 說「即使方法簡單，你還是要說清楚假設和限制；這才是技術寫作的核心」 |
| 「限制要怎麼想？我不知道我的方法有什麼問題」 | 用三個框架引導：（1）你的模型假設了什麼？（2）你的數值方法在什麼條件下不準確？（3）你的參數怎麼來的，若不同會怎樣？ |
| 學生寫出的是操作手冊而非方法章 | 請他刪掉所有「先打開…然後…最後…」的結構，改以「本研究採用……」的學術語氣重寫 |
| 學生不知道進階簡報和 Level 2 簡報差在哪 | 說「Level 2 是講給父母聽的，Level 3 是講給大學教授聽的；後者需要更多技術細節和誠實的限制討論」 |
