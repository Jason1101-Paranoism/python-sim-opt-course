# Level 2 週 02 作業單

**截止時間**：下次上課前（週 03 group 課開始前）

**目標**：把課堂排隊範例改成你自己題目的版本，或至少寫出 simulate_once 的架構草稿。

---

## 要繳交的東西

- [ ] 一份「我的模型草圖」（手寫 / 打字皆可）
- [ ] 修改後的程式（`.py` 或 `.ipynb`），或 simulate_once 的偽程式碼

---

## 具體要求

### 任務 A（必做）：完成你的模型草圖

填寫以下四個元素（以你的專題方向為主，不需要是完整版本）：

1. **狀態**：系統在某個時刻，你需要追蹤什麼？
2. **參數**：你打算調整哪些數值來做實驗？（至少列出 2 個）
3. **隨機輸入**：你的模型中有哪些不確定性？（若無隨機性也要說明）
4. **輸出指標**：你的 simulate_once 最後要回傳什麼？

### 任務 B（必做）：跑一次排隊模擬的變體

修改課堂範例中的至少一個地方，跑出結果，截圖並說明：
- 你改了什麼
- 結果和你預期的一樣嗎？有沒有讓你意外的地方？

**（允許修改的方向）**：
- 加入第二個服務台（需要修改 simulate_once 邏輯）
- 讓服務時間也是隨機的（用 `random.expovariate`）
- 改變 `rates` 的範圍，看看圖的形狀怎麼變

### 任務 C（選做，有興趣才做）：

試著把你自己的題目寫成 simulate_once 的開頭（只需要函數定義 + 輸入輸出 + 幾行骨架，不需要完整實作）：

```python
def simulate_once(參數1, 參數2, ...):
    """
    說明這個函數模擬什麼
    假設：...
    回傳：...
    """
    # 你的骨架
    pass
```
