# 學術誠信政策

## 核心原則

> 你繳交的所有作業、程式、報告，必須是你自己能理解、能解釋、能修改的。

這門課的目標不是讓學生「交出一份漂亮的東西」，而是讓學生真正學會「從問題到研究的完整流程」。交出一份不屬於自己的作品，對學生本人的成長沒有任何意義。

---

## 什麼是學術誠信違規

### 明確違規行為

| 行為 | 說明 |
|------|------|
| 抄襲他人程式 | 複製其他學生或網路上的程式，未做實質修改即繳交 |
| 抄襲他人報告 | 複製他人（含網路文章）的文字段落，未加引用 |
| AI 全文代寫 | 用 AI 生成整份報告或整個程式，自己未理解內容（詳見 `ai-usage-policy.md`） |
| 代勞他人作業 | 幫其他學生做他們的作業或報告 |
| 偽造資料或結果 | 捏造模擬數據或圖表 |

### 灰色地帶的處理原則

| 情況 | 判斷方式 |
|------|----------|
| 參考網路範例程式後改寫 | 可接受，但需在程式註解中標明來源 |
| 和同學討論後各自撰寫 | 可接受，討論是學習的一部分 |
| 同學合作寫程式，一起繳交 | 只有在明確標示為「小組作業」的週次才可接受 |
| 用 AI 修改語氣、潤飾文字 | 可接受，但核心論述須是學生自己的 |

---

## 引用規範

### 程式碼引用

若程式中有參考他人（網路、教材以外的來源）的邏輯，須在程式中以註解標明：

```python
# 參考來源：[說明來源，如 Stack Overflow 連結或同學名字]
# 我的修改：[說明你做了什麼修改]
```

### 報告文字引用

- Level 1–2：不強制學術引用格式，但不可直接複製文字
- Level 3：有使用外部資料或觀念，需標明來源（簡易格式即可）
- Level 4：須使用標準引用格式（APA 或 IEEE，依老師指定）

## 違規處理程序

### 第一級：疑似違規（主動處理）

**觸發條件**：作業或報告中出現學生在 1-on-1 中無法解釋的內容。

**處理**：
1. 老師在 1-on-1 中以提問方式確認（「你能說說這段程式為什麼這樣設計？」）
2. 若學生說不出來，請其在一週內重做該部分
3. 不通知家長，不記錄為正式違規

---

### 第二級：確認違規（輕度）

**觸發條件**：明確發現局部抄襲（如一個函數、一段落），但整體作業有個人努力。

**處理**：
1. 1-on-1 中直接說明發現的問題
2. 請學生重做違規部分，一週內繳交
3. 口頭通知家長（說明情況與改善方式）
4. 記錄在學生紀錄表中

---

### 第三級：嚴重違規

**觸發條件**：整份作業或報告為抄襲、AI 全文代寫、偽造資料。

**處理**：
1. 書面通知家長（說明事件與後果）
2. 該份作業標記為「未完成」，不接受補交
3. 若發生在期末報告：期末評分不計，需重做或降級
4. 若使用成果於校外申請（AP/IB/競賽），提醒家長學校可能有額外的後果

---

## 對家長的溝通原則

- 第一、二級違規：以改善學習方式為主軸，不上綱到「品格問題」
- 第三級違規：說明事實，給出明確後果，並提供補救方向
- 任何情況下，不在群組或公開場合討論個別學生的違規情形

---

## 預防優先

最有效的誠信保護是**在 1-on-1 中讓學生真正理解自己的作品**。

老師的責任：
- 每週 1-on-1 確認學生能解釋自己的程式與報告
- 在作業說明中清楚描述「允許」與「不允許」的行為
- 對「交出漂亮作品但說不清楚」的學生，優先提供支持而非懲處
