# Level 3 週 08 作業單：專題最佳化流程 + 敏感度分析

## 作業名稱與目標

**作業名稱**：高級專題程式 v2.0（加入最佳化）

**目標**：
在上週的核心模型基礎上，加入完整的最佳化流程和至少一項敏感度分析，並用 3–5 句話說明最佳解的意義與合理性。

---

## 繳交清單

| 項目 | 格式 | 說明 |
|------|------|------|
| 主程式（含最佳化） | `.py` 檔 | 必須可執行 |
| 目標函數視覺化圖 | `.png` | 畫出決策變數 vs 目標函數值的關係 |
| 最佳化結果圖或表 | `.png` 或程式輸出截圖 | 顯示最佳解的值和對應的目標值 |
| 敏感度分析圖 | `.png` | 至少一張，一個參數的變化對最佳解的影響 |
| 結果說明（文字） | 程式注釋中或獨立 `.txt` | 3–5 句話說明最佳解是什麼、為什麼合理、有什麼限制 |

**截止時間**：下次 group 課前（週 09 上課前 24 小時）

---

## 具體要求

1. **目標函數必須正確定義**：回傳一個純量數字，且方向正確（最小化 = 越小越好）
2. **需呼叫 SciPy 最佳化函數**：`minimize`、`minimize_scalar`、`differential_evolution` 任一皆可
3. **需說明初始點選擇**：在程式注釋中說明你的 x0 是怎麼選的，試過幾個初始點
4. **敏感度分析需有對照**：至少 5 個不同的參數值，畫出趨勢圖
5. **結果說明必須具體**：「最佳接種比例為 67%，代表需要至少三分之二的人口接種才能讓感染高峰低於 100 人」（這種程度）

---

## 自我檢查清單

- [ ] 目標函數寫好了，可以手動呼叫，輸出一個合理的數字
- [ ] `minimize` 或 `minimize_scalar` 成功執行，有輸出 `result.x` 和 `result.fun`
- [ ] 確認最佳化結果合理（用人工判斷，不是只看程式輸出就信任）
- [ ] 試過至少 2 個不同的初始點（x0），確認結果一致或記錄差異
- [ ] 畫了目標函數的視覺化圖（決策變數 vs 目標值）
- [ ] 畫了敏感度分析圖（一個參數改變時，最佳解如何變化）
- [ ] 結果說明：有說「最佳值是多少」「對應什麼含義」「這個解的限制是什麼」

---

## 結果說明範例

**不夠好的說明**：「最佳化結果顯示最佳值是 0.67。」

**好的說明**：
「在 β=0.3、γ=0.1、N=10000 的條件下，最佳接種比例為 67%。這意味著在我們的模型假設（均勻混合、接種後完全免疫）下，至少需要三分之二的人口接種，才能讓感染高峰降到 200 人以下（佔總人口 2%）。這個結果與群體免疫閾值的理論公式 1 - 1/R0 = 1 - 1/(0.3/0.1) = 67% 一致，說明我們的數值結果合理。限制：實際疫情中接觸不是均勻的，真實閾值可能更高或更低。」

---

## 常見失分項目

| 問題 | 說明 |
|------|------|
| 目標函數回傳陣列而非純量 | 回傳前加 `.max()` 或 `[0]` 等取出單一數字 |
| 只跑一個初始點 | 局部解的問題必須由多個初始點的比較來說明 |
| 敏感度分析只有數字，沒有圖 | 圖比表格更容易看出趨勢，這週必須有圖 |
| 結果說明只複製 print 輸出 | 說明需要你的解讀，不是程式幫你說 |
