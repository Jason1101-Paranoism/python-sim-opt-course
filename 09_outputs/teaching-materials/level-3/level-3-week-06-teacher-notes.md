# Level 3 週 06：高級專題選題與方法設計（教師教案）

## 本週學習目標

1. 學生能說清楚自己要做的「高級專題」是什麼問題，以及為什麼需要數值方法或最佳化
2. 學生能為自己的專題設計出可執行的方法計畫（用哪些工具、什麼數值手法、哪些實驗）
3. 學生能在 5 分鐘內清楚地把專題方向表達給同儕，並接受具體回饋
4. 老師能掌握全班題目方向，確認每位學生的題目難度與工具選擇是否合理

---

## 本週課堂結構（中期展示週——特殊結構）

> 本週是中期展示週，取消範例示範與學生實作，改為展示與回饋流程。

| 段落 | 時間 | 內容 |
|------|------|------|
| 開場 + 說明流程 | 10 分鐘 | 老師說明今天流程與回饋格式，學生確認「方向摘要」準備好 |
| 學生展示（每人 5 分鐘） | 約 40 分鐘（4–8 人） | 每位學生口頭說明自己的高級專題方向 |
| 同儕 + 老師即時回饋 | 每人 5 分鐘，穿插展示後 | 同儕先說、老師補充，重點：問題清不清楚？方法合不合理？ |
| 老師總結共同問題 | 15 分鐘 | 整理全班共同弱點、常見選題錯誤，給出下週重點提示 |
| 作業說明 + 收尾 | 15 分鐘 | 說明本週作業（完整一頁高級專題計畫書）、預告下週實作開始 |

> 若班級人數 4 人：展示 + 回饋共 40 分鐘；若 8 人：共 80 分鐘，需壓縮每人回饋至 3 分鐘。

---

## 各段落執行細節

### 開場 + 說明流程（10 分鐘）

**老師開場話術建議**：

> 「今天是 Level 3 的中期展示。前五週我們學了 NumPy、SciPy、ODE、最佳化——今天你們要告訴我，你們打算把這些工具用在哪裡、怎麼用。這不是成品展示，是方向確認。你說得越清楚，我給的建議就越有用。」

**說明回饋規則**：
- 每人展示後，同儕先說「我聽到的問題是……」（確認理解）
- 再說「我有一個問題想問……」（具體問題，不是「很好」）
- 老師最後補充技術面建議

**黑板／白板先寫好評分框架**（展示時讓同儕對照）：
```
1. 問題是什麼？（明確？可用數值方法處理？）
2. 要用什麼工具？（具體？NumPy / SciPy 哪個函數？）
3. 預計什麼實驗？（有可比較的條件嗎？）
4. 最難的地方是哪裡？（學生有沒有預估到風險？）
```

---

### 學生展示（每人 5 分鐘）

**老師注意事項**：
- 計時嚴格，5 分鐘到就打斷（練習說重點）
- 每位學生展示前，老師先說「請用一句話說你的問題」——若說不出來，就是題目不清楚
- 老師在每人展示時，在自己的紀錄表上填寫：

| 學生 | 題目 | 擬用工具 | 最大風險 | 我的建議 |
|------|------|----------|----------|----------|
| （姓名） | | | | |

**常見題目方向（可預期的，提前備一些回饋語）**：
- 傳染病 SIR 模型升級版 → 提醒：參數估計是真正的難點，建議先 fix 參數做敏感度
- 彈道／物理模擬 → 提醒：ODE 步長的選擇會影響精度，需要收斂測試
- 排程最佳化 → 提醒：目標函數需要明確，決策變數要具體（連續？離散？）
- 機器學習 → 建議轉向：Level 3 重點是數值方法，ML 不在本課範疇，可以縮小為「最佳化超參數」

---

### 老師總結共同問題（15 分鐘）

**固定總結框架**（每次中期展示都用這個）：

**共同強項**（先說好的，2 分鐘）：
> 「今天大部分人的問題都有具體的數學模型，這是好的開始。」

**共同弱點（通常 2–3 個）**：

1. **問題太大**：「我要模擬整個氣候系統」→ 縮小到一個變數、一個時間段
2. **工具選擇不清楚**：「我會用 SciPy」→ 要說是哪個函數（`scipy.integrate.solve_ivp`？`scipy.optimize.minimize`？）
3. **實驗設計缺乏對照**：「我要跑模擬看結果」→ 要說「固定 A，改變 B，看 C 如何變化」

**老師下週提示**：
> 「下週開始動手寫程式。你的第一個 milestone 是：讓你的模型可以跑起來、輸出一張圖。不需要完美，但要可以執行。」

---

### 作業說明（10 分鐘，含在收尾段）

**本週作業**：完整版「高級專題計畫書」（一頁，使用指定格式）

強調：
- 今天展示的是草稿，這份作業是修改後的正式版本
- 下週老師會根據這份計畫書給逐人回饋
- 格式要求：問題陳述、模型說明、擬用工具（具體函數名）、預計實驗（有對照條件）、預計最大困難

## 常見問題與處理方式

| 問題 | 處理方式 |
|------|----------|
| 學生說「我還沒想好題目」 | 請他先說「我最感興趣的領域是 X」，老師當場幫他縮小一個可做的問題 |
| 學生的題目和 Level 2 完全一樣，沒有升級 | 問「你打算在哪裡加入數值方法？」，若加不進去，請他考慮不同方向 |
| 學生想用的工具不在 Level 3 範疇（如 PyTorch） | 說明 Level 3 的工具限制，引導回 SciPy 框架；若真的需要，可以做 1-on-1 特別討論 |
| 同儕回饋流於「很好」「不錯」 | 老師示範一次具體回饋：「你說的目標函數是什麼？我不清楚你要最小化什麼」 |
| 展示超時（學生講不完） | 5 分鐘到就打斷，說「時間到，我問你一個問題：你的問題是什麼？」讓他用一句話回答 |
| 班級人數少（4 人），時間很充裕 | 每人回饋時間延長至 8 分鐘，老師可更深入問技術細節；多留 10 分鐘做非正式討論 |
