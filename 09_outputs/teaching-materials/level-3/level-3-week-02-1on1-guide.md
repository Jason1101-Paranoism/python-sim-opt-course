# Level 3 週 02 — 1-on-1 討論指南
## 主題：確認 NumPy 改寫進度，找出難點

---

## 本週個別討論聚焦

本週 1-on-1 的核心任務：

1. **確認學生的 NumPy 改寫是否成功**：不只是「能跑」，而是真的理解向量化
2. **找出學生舊專題的技術卡點**：有些計算因為累積依賴而難以向量化，需要個別指導策略
3. **為 Week 3 SciPy 做準備**：根據學生的專題，引導他思考「哪裡需要解方程或積分」

---

## 建議討論流程（60 分鐘）

| 段落 | 時間 | 內容 |
|------|------|------|
| 回顧 | 10 分鐘 | 學生展示 NumPy 改寫結果（計時比較） |
| 主要討論 | 35 分鐘 | 深入討論改寫邏輯 + 找出下一步升級點 |
| 下週目標 | 10 分鐘 | 確認 Week 3 SciPy 的準備方向 |
| 策略備註 | 5 分鐘 | 教師記錄學生進展 |

## 具體問題清單

### 段落一：回顧（10 分鐘）

請學生打開作業，展示兩個版本的計時結果：

1. 「你的加速倍率是多少？比你預期的高還是低？」
2. 「你用了哪些 NumPy 函數來取代 for 迴圈？」（讓他說，而不是你說）
3. 「結果驗證是 True 嗎？如果不是，你找出原因了嗎？」

**觀察重點**：
- 加速效果明顯嗎？（如果只快 2–3 倍，通常代表有部分運算仍在 Python 層）
- 學生能解釋為什麼 NumPy 更快嗎？（測試概念理解）
- 驗證步驟做了嗎？（很多學生只做改寫，不做驗證）

---

### 段落二：主要討論（35 分鐘）

**Part A：深入討論改寫邏輯（15 分鐘）**

4. 「你的 for 迴圈改寫的過程中，有沒有哪一步讓你卡住了？」
5. 「你有沒有嘗試過先把整個迴圈的輸入資料改成 array？那一步是怎麼做的？」
6. 「如果我給你一個新的計算：`[x**2 + sin(x) for x in range(1000)]`，你怎麼用 NumPy 改寫？」（快速測試）

**如果學生無法回答第 6 題**：
→ 一起做，邊做邊解釋 `np.arange`、向量運算、`np.sin`
→ 說明這是 NumPy 的基本模式，後面每週都會用到

**Part B：找出下一步升級點（20 分鐘）**

7. 「你的舊專題中，有沒有「想找到一個 x 讓某個式子等於零」的情況？」
   （例如：找到讓供需平衡的價格、找到讓誤差最小的參數）
8. 「有沒有要計算某個連續量的「總和」？不是加法，是積分那種？」
   （例如：某個函數的面積、機率分布的累積）
9. 「你的模型有沒有隨時間演變的動態過程？」

**根據回答判斷 Week 3 方向**：
- 如果學生有「找零點」的需求 → Week 3 重點：`brentq` 求根
- 如果學生有「計算面積/總量」 → Week 3 重點：`quad` 積分
- 如果學生兩者都沒有 → 用課程內建的範例（彈簧振動、族群模型）先做練習

---

### 段落三：下週目標（10 分鐘）

10. 「下週我們要學 SciPy 的求根和積分。你的專題裡，最有可能用到哪一個？」
11. 「我希望你在下週上課前，嘗試自己執行一次：`from scipy.optimize import brentq; result = brentq(lambda x: x**2 - 2, 0, 2); print(result)`，看看結果是多少。」

**確認 Week 3 的個人化目標**：

根據本次討論，幫學生設定一個具體的 Week 3 個人目標，例如：
- 「找到你的族群模型中『達到穩態的時間點』，用 `brentq` 求出來」
- 「計算你的物理模型在 0 到 10 秒的總能量，用 `quad` 積分」

## 本週學生應達成的里程碑

- [ ] 完成 NumPy 改寫作業（兩個版本 + 計時 + 驗證）
- [ ] 能口頭解釋「向量化為什麼比 for 迴圈快」（不需要完整技術解釋，有正確直覺即可）
- [ ] 找到至少一個「下週可以用 SciPy 處理的計算」（可以是求根、積分或任一）
- [ ] Python 環境中已安裝 SciPy（`pip install scipy`），確認可以匯入

**如果學生達不到里程碑**：
- NumPy 改寫沒成功 → 在 1-on-1 中一起完成一個最小版本，讓他帶回去繼續
- 找不到 SciPy 的應用場景 → 先不勉強，Week 3 直接用課程範例，後面再對應到個人專題
