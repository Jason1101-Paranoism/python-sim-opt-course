# Level 3 週 07：1-on-1 討論指南

## 本週個別討論聚焦

本週 1-on-1 的核心是**程式除錯與架構確認**。學生剛開始寫專題程式，最常見的問題是：
1. ODE 函數的簽名寫錯（`solve_ivp` 要求 `f(t, y, *args)` 格式）
2. 收斂測試不知道怎麼解讀
3. 結果圖「看起來奇怪」但不知道哪裡錯

這次 1-on-1 要確保每位學生的程式可以執行，並幫他們建立「我能自己除錯」的能力。

---

## 建議討論流程（60 分鐘）

| 時段 | 時間 | 內容 |
|------|------|------|
| 回顧 | 10 分鐘 | 學生展示本週寫的程式；老師快速 code review |
| 主要討論：程式除錯與架構 | 35 分鐘 | 針對卡點逐一處理，確認數值方法正確整合 |
| 下週目標 | 10 分鐘 | 確認下週最佳化的入場條件：這週程式必須先跑起來 |
| 策略備註 | 5 分鐘 | 時程、競賽目標 |

---

## 具體問題清單

### 回顧段（10 分鐘）

1. 「讓我看一下你這週寫的程式。先跑給我看。」
   - 若程式可以跑：「結果圖看起來合理嗎？你怎麼知道？」
   - 若程式報錯：先看錯誤訊息，引導學生自己解讀（見下方除錯流程）

2. 「你做了收斂測試嗎？結果怎樣？」

3. 「你做了邊界案例嗎？符合預期嗎？」

---

### 主要討論段（35 分鐘）

**情境 A：程式可以執行，但結果看起來奇怪（最常見）**

提問序列：
- 「你的模型在 t=0 時，初始條件符合你的設定嗎？」
  （請學生 `print(sol.y[:, 0])` 確認）
- 「你有沒有試過把一個參數設為 0，看結果是否符合預期？」
- 「你的 ODE 函數回傳值的順序和你的 y0 順序一致嗎？」
  （例如：`y0 = [S, I, R]`，ODE 函數要回傳 `[dS/dt, dI/dt, dR/dt]`，順序不能顛倒）

---

**情境 B：solve_ivp 報錯**

常見錯誤與提示：

| 錯誤訊息 | 原因 | 提示 |
|----------|------|------|
| `ValueError: y0 must be 1-D` | y0 格式錯誤（如傳了 2D array） | 確認 `y0 = [S0, I0, R0]`，不是 `[[S0, I0, R0]]` |
| `TypeError: fun() missing required argument` | ODE 函數參數不對 | 確認函數簽名是 `def model(t, y, *extra_params)` |
| `solve_ivp failed` + 步數訊息 | 步長過大，系統太 stiff | 試試 `method='Radau'` 或 `method='BDF'`（處理 stiff ODE） |
| 結果出現 `nan` 或 `inf` | 計算過程出現除以零 | 檢查分母是否可能為 0（如 N=0 或某個狀態跑到負數） |

---

**情境 C：學生的問題不是 ODE 而是靜態最佳化**

- 本週目標調整為：把目標函數和限制條件寫清楚
- 確認 `scipy.optimize.minimize` 的呼叫格式
- 問：「你的目標函數輸入是什麼？輸出是什麼？能不能直接呼叫 `minimize`？」

---

**問題四：收斂測試的討論（視情況需要 5–10 分鐘）**

- 「你的收斂測試顯示什麼？步長從 5.0 到 0.01 時，結果差多少？」
- 「你認為用 max_step = 1.0 對你的題目夠精確嗎？」
- 「如果結果不收斂（不同步長差很多），可能是什麼原因？」
  → 提示：系統可能是 stiff ODE，需要換方法

---

**問題五：下週最佳化的入場條件確認**

- 「你的程式現在輸出什麼？是一個數字還是一張圖？」
- 「下週你要最佳化哪個目標？（最大化感染高峰？最小化感染總人數？）」
- 「你的目標可以用你現在的模型直接計算出來嗎？」

如果學生這週程式還沒跑起來 → 下週 1-on-1 要先補完這週任務，最佳化往後延

---

### 下週目標確認（10 分鐘）

**下週 Milestone（週 08）**：

```
在現有模型基礎上，加入最佳化流程：
- 定義目標函數（把模型輸出的某個指標轉成 optimize 可以最小化的值）
- 用 scipy.optimize.minimize 找最佳參數
- 畫出「參數值 vs 目標值」的圖（敏感度分析的起點）
```

幫學生填入下週具體要做的事：
- 「你的目標函數是：___」
- 「你要最佳化的參數是：___」
- 「你預計用的 SciPy 函數是：`scipy.optimize.___`」

## 本週學生應達成的里程碑

| 里程碑 | 驗收方式 |
|--------|----------|
| 主程式可執行，不報錯 | 老師現場跑或看繳交的截圖 |
| 有用 SciPy 數值函數 | 程式中可以看到 `from scipy import...` 並有實際呼叫 |
| 輸出至少一張圖 | 圖檔一起繳交 |
| 完成收斂測試 | 繳交的程式中有收斂測試的輸出 |
| 完成至少 2 個邊界案例 | 程式中有邊界案例的測試程式碼 |

## 除錯輔助工具（教師備用）

若學生的 ODE 函數不確定對不對，可以用這個快速檢查：

```python
# 快速檢查 ODE 函數：在 t=0 時，回傳值是否合理
y0_check = [9990, 10, 0]
t_check = 0
params_check = (0.3, 0.1, 10000)
result = my_model(t_check, y0_check, *params_check)
print("t=0 時的各狀態變化率：", result)
# 對 SIR 模型而言：
# dS/dt 應該是負數（易感者在減少）
# dI/dt 應該是正數（感染者在增加，假設 R0 > 1）
# dR/dt 應該是正數（康復者在增加）
```
