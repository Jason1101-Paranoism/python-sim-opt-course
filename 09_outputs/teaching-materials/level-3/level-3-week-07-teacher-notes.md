# Level 3 週 07：專題實作 — 模型與數值方法（教師教案）

## 本週學習目標

1. 學生能將前幾週學的數值方法（ODE solver 或數值積分）整合進自己的專題程式中
2. 學生能說明「收斂」的概念，並透過改變步長或容忍度來檢查結果穩定性
3. 學生能用至少一種直觀方式確認計算結果合理（邊界案例比較、與簡化版本對照）
4. 學生手上有一個可執行的專題程式，產出有意義的圖表

### 概念講解（25 分鐘）

#### 一、收斂是什麼？（10 分鐘）

**生活例子引入**（3 分鐘）：

> 「你用 Google Maps 計算從台北到台中的距離，答案是 175 公里。如果你每隔 10 公里取一個點來估算總距離，你得到 174 公里；如果每隔 1 公里取一個點，你得到 174.8 公里；每隔 0.1 公里，174.9 公里。結果越來越接近真實值——這就叫收斂。」

**數值方法的收斂**（5 分鐘）：
- ODE solver 的步長（`max_step`）越小 → 結果越精確，但計算越慢
- 數值積分的區間數越多 → 結果越精確
- 我們要找到「夠精確、又不太慢」的平衡點
- 收斂測試：用不同步長跑，看結果是否趨近同一個值

**白板圖示**：畫出「步長 vs 誤差」的 L 型曲線（步長太大誤差大，步長太小誤差趨近零）

---

#### 二、如何確認結果合理？（10 分鐘）

**三種驗證方法**：

1. **邊界案例**：把參數設到極端值，看結果是否符合直覺
   - SIR 模型：若接觸率 β = 0，感染者應該永遠不增加
   - 若結果不符合，代表程式有 bug

2. **與簡化版本對照**：先用 for 迴圈做差分（簡單版），再用 ODE solver，比較兩者
   - 若差距合理（取決於步長），說明兩個程式都對
   - 若差距很大，先檢查初始條件是否一致

3. **量綱檢查**：確認輸出的單位和數量級是否合理
   - SIR 模型：感染者 I 不能超過總人口 N
   - 物理模擬：速度不能超過光速（除非你在模擬相對論）

**快速問答**（3 分鐘）：
- 「如果 SIR 模型的感染人數算出來是負數，你怎麼辦？」
- 「如果把步長從 1 天改成 0.01 天，結果幾乎一樣，這代表什麼？」

---

#### 三、整合數值方法的程式架構（5 分鐘）

**標準架構（畫在白板）**：

```
1. 定義模型（ODE 函數 or 目標函數）
2. 設定初始條件和參數
3. 呼叫 SciPy 函數
4. 解讀輸出格式

```

強調：「今天的目標是讓你的程式走完這六步。不需要完美，只要可以執行。」

### 範例示範（Live coding，30 分鐘）

**示範主題**：SIR 傳染病模型 + 收斂測試

從空白檔案開始，邊打邊說明。

```python
# 匯入必要的函式庫
import numpy as np
import matplotlib.pyplot as plt
from scipy.integrate import solve_ivp

# ============================================================
# 第一步：定義模型（ODE 函數）
# ============================================================
def sir_model(t, y, beta, gamma, N):
    """
    SIR 傳染病模型的 ODE 方程式。
    
    參數：
        t     : 時間（solve_ivp 會自動傳入，我們不需要直接用它）
        y     : 狀態向量 [S, I, R]
        beta  : 接觸率（每天每個感染者能傳染幾個易感者）
        gamma : 康復率（1/gamma 是平均感染天數）
        N     : 總人口數
    
    回傳：各狀態的變化率 [dS/dt, dI/dt, dR/dt]
    """
    S, I, R = y          # 拆解狀態向量
    dS_dt = -beta * S * I / N
    dI_dt =  beta * S * I / N - gamma * I
    dR_dt =  gamma * I
    return [dS_dt, dI_dt, dR_dt]

# ============================================================
# 第二步：設定參數和初始條件
# ============================================================
N = 10000        # 總人口
I0 = 10          # 初始感染者
R0_count = 0     # 初始康復者（注意：這裡叫 R0_count，不要和基本再生數 R0 搞混）
S0 = N - I0 - R0_count  # 初始易感者

y0 = [S0, I0, R0_count]  # 初始狀態向量

beta = 0.3    # 接觸率
gamma = 0.1   # 康復率
t_span = (0, 160)  # 模擬時間範圍（天）

# ============================================================
# 第三步：呼叫 solve_ivp 求解 ODE
# ============================================================
# solve_ivp 的主要參數：
#   fun    : 我們定義的 ODE 函數
#   t_span : 時間範圍 (t_start, t_end)
#   y0     : 初始條件
#   args   : 傳入 ODE 函數的額外參數（tuple 格式）
#   dense_output : 設為 True 可以在任意時間點查詢值
sol = solve_ivp(
    fun=sir_model,
    t_span=t_span,
    y0=y0,
    args=(beta, gamma, N),
    dense_output=True,
    max_step=1.0   # 最大步長：1天（稍後我們會改這個值做收斂測試）
)

# ============================================================
# 第四步：解讀輸出 + 畫圖
# ============================================================
# sol.t 是 solver 選擇的時間點陣列
# sol.y 是各狀態的值，shape = (3, 時間點數)
t_eval = np.linspace(0, 160, 500)  # 產生均勻的時間點用於畫圖
y_eval = sol.sol(t_eval)           # 在這些時間點查詢解

plt.figure(figsize=(10, 5))
plt.plot(t_eval, y_eval[0], label='S（易感者）', color='blue')
plt.plot(t_eval, y_eval[1], label='I（感染者）', color='red')
plt.plot(t_eval, y_eval[2], label='R（康復者）', color='green')
plt.xlabel('時間（天）')
plt.ylabel('人數')
plt.title(f'SIR 模型（β={beta}, γ={gamma}, N={N}）')
plt.legend()
plt.tight_layout()
plt.savefig('sir_model.png', dpi=150)
plt.show()
print(f"感染高峰：{y_eval[1].max():.0f} 人，在第 {t_eval[y_eval[1].argmax()]:.1f} 天")

# ============================================================
# 第五步：收斂測試（改變 max_step，看結果是否穩定）
# ============================================================
peak_values = {}
for max_step in [5.0, 1.0, 0.5, 0.1, 0.01]:
    sol_test = solve_ivp(
        fun=sir_model,
        t_span=t_span,
        y0=y0,
        args=(beta, gamma, N),
        dense_output=True,
        max_step=max_step
    )
    y_test = sol_test.sol(t_eval)
    peak = y_test[1].max()
    peak_values[max_step] = peak
    print(f"max_step={max_step:.3f} 天 → 感染高峰 = {peak:.2f} 人")

# ============================================================
# 第六步：邊界案例驗證
# ============================================================
# 測試：beta = 0 時，感染者應保持不變
sol_zero_beta = solve_ivp(
    fun=sir_model,
    t_span=t_span,
    y0=y0,
    args=(0.0, gamma, N),   # beta = 0，沒有傳播
    dense_output=True
)
y_zero = sol_zero_beta.sol(t_eval)
print(f"\n邊界案例（β=0）：感染者最大值 = {y_zero[1].max():.2f}，最小值 = {y_zero[1].min():.2f}")
print("預期：感染者應只減少（因為 γ > 0），不增加 ✓")
```

**Live coding 過程中的刻意錯誤**（讓學生找）：
- 在 `sir_model` 函數中，把 `dS_dt = -beta * S * I / N` 寫成 `+beta * S * I / N`（符號錯誤）
- 讓學生觀察圖的形狀，問「S 曲線應該往上還是往下？為什麼？」

### 回顧與預告（20 分鐘）

**本週三個關鍵點**（白板寫下）：

## 常見問題與處理方式

| 問題 | 處理方式 |
|------|----------|
| 「我的問題不是 ODE，我不知道要用什麼」 | 確認是靜態最佳化 → 今天先把目標函數定義好，下週用 minimize |
| `solve_ivp` 輸出的 `sol.t` 時間點不均勻 | 解釋：solver 自動選步長，要均勻畫圖請用 `dense_output=True` + `sol.sol(t_eval)` |
| 程式跑很慢 | 先檢查 `max_step` 是否設太小；另外檢查 `t_span` 是否太長 |
| 學生想加很多功能，但核心還沒做好 | 說「先讓最簡單版本跑起來，今天只做核心，下週才加功能」 |
| 結果圖出現負數（如負人口） | 可能是步長太大，ODE 積分過衝；縮小 `max_step` 通常可以解決 |
