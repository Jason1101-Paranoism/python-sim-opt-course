# Level 2 週 03：題目收斂與 RQ 寫法

## 一、什麼是好的 Research Question？

RQ（Research Question，研究問題）是你整個專題的起點。
RQ 寫得清楚，後面的模型、程式、報告才有方向。

好的 RQ 要滿足三個條件：

### 條件 1：可量化

你要研究的問題，最後的答案是一個**數字或可以量化的現象**。

| 不好的例子 | 為什麼不好 | 改寫後 |
|-----------|-----------|--------|
| 「病毒怎麼傳播？」 | 「怎麼」太廣，無法量化 | 「接觸率增加 2 倍時，感染高峰的人數增加多少？」 |
| 「哪種策略比較好？」 | 「比較好」是主觀的 | 「策略 A 的平均獲利是策略 B 的幾倍？」 |

### 條件 2：可用模擬回答

你的問題可以**用一個程式模型**來探索和回答。

- 問「真實世界中，台北的交通問題怎麼解決？」→ 需要真實資料，不適合模擬
- 問「在簡化的十字路口模型中，紅燈時長如何影響等待時間？」→ 可以模擬 ✅

> 關鍵：你不是在重現真實世界，而是在研究一個**簡化模型**的行為。

### 條件 3：範圍具體

你的問題要**縮小到你能完成的範圍**，不是研究整個系統。

| 太大的問題 | 收斂後的 RQ |
|-----------|-----------|
| 「模擬氣候變化對生態的影響」 | 「在捕食者—獵物模型中，溫度升高（影響繁殖率）如何改變族群動態？」 |
| 「研究股市崩盤」 | 「在隨機漫步股價模型中，何種波動率下，初始資金 1000 元的投資者在 100 天內破產的機率超過 50%？」 |

---

## 二、RQ 寫作模板

```
在 [模型 / 情境描述] 中，
當 [你要改動的參數] 從 [範圍起點] 變化到 [範圍終點] 時，
[你要觀察的輸出指標] 如何改變？
```

**使用範例：**
> 在單服務台排隊模型中，當顧客到達率從 0.5 人/分鐘增加到 1.5 人/分鐘（服務時間固定為 1 分鐘）時，平均等待時間如何變化？

## 三、改寫練習

把以下模糊問題，用 RQ 模板改寫成好的 RQ：

**原始問題 1**：「研究賭博策略」

我的 RQ：_______________________________________

**原始問題 2**：「模擬人口成長」

我的 RQ：_______________________________________

**原始問題 3**：（用你自己的題目方向）

我的 RQ：_______________________________________

## 四、RQ 自我檢查表

寫出你的 RQ 後，用以下清單確認：

| 檢查點 | ✅ / ❌ |
|--------|--------|
| RQ 最後的答案是一個數字或可量化的現象 | |
| 這個問題可以用一個簡化的模擬程式回答 | |
| 問題有指出「改變什麼」和「觀察什麼」 | |
| 問題範圍不需要超過 4 週的程式開發 | |
| 不需要外部真實資料集就能回答 | |
