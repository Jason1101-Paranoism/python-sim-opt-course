# Level 1 週 1：什麼是模擬？骰子實驗
## 教師教案（Teacher Notes）

---

### 本週學習目標

1. 學生能說出「模擬」與「直接計算」的差異
2. 學生能寫出一個擲骰子 N 次並統計結果的 Python 程式
3. 學生能用直方圖呈現模擬結果，並解讀圖表

---

### 課堂時間分配（120 分鐘）

| 段落 | 時間 | 內容 |
|------|------|------|
| 暖身回顧 | 10 分鐘 | 自我介紹 + 課程說明 + 提問：「你覺得模擬是什麼？」 |
| 概念講解 | 25 分鐘 | 模型 vs 現實、隨機 vs 確定、重複試驗的意義 |
| 範例示範 | 30 分鐘 | Live coding：擲骰子模擬程式從零開始 |
| 學生實作 | 35 分鐘 | 學生自己完成骰子模擬並畫直方圖 |
| 回顧與預告 | 20 分鐘 | 總結 3 個關鍵點、說明本週作業、預告下週 |

### 暖身（10 分鐘）

**第一週特殊流程**：

1. 老師自我介紹（2 分鐘）：名字、背景、為什麼開這門課
2. 課程整體說明（3 分鐘）：
   - 這堂課不考試，但每週都要有產出
   - 10 週後每個人都會有一個「自己的模擬專題」
   - 1-on-1 是做什麼的（個人題目、個人 debug）
3. 開場提問（5 分鐘）：「你聽過『模擬』這個詞嗎？在哪裡聽過？」
   - 讓每個學生說 1 句，不評對錯
   - 預期答案：電腦遊戲、天氣預報、飛行模擬器、股票回測

---

### 概念講解（25 分鐘）

**引入（3 分鐘）**

> 「如果我問你：擲一個公正骰子 1000 次，6 點會出現幾次？」

讓學生喊答案。通常會說「大約 167 次」或「1/6 的 1000 次」。
接著說：「你怎麼知道？你有算過嗎？還是你用了某個規則？」

**核心概念（15 分鐘）**

寫在白板上：

```
現實世界（複雜、有噪音、不可控）
        ↓ 簡化、假設
模型（規則 + 隨機）
        ↓ 用電腦跑很多次
模擬結果（可以統計、可以分析）
```

說明三個關鍵詞：
- **模型**：對現實的簡化版本，「我假設骰子是公正的」就是一個假設
- **隨機**：不是亂來，而是有機率分佈的不確定性
- **重複試驗**：跑一次沒意義，跑 1000 次才能看出規律

**小問答確認理解（7 分鐘）**

- 「天氣預報是模擬嗎？」→ 是，它也是跑很多次模型，看哪個結果最常出現
- 「如果我只擲骰子 10 次，6 點一次都沒出現，骰子是作弊的嗎？」→ 不一定，樣本數太少

### 範例示範（30 分鐘）

**從空白檔案開始，邊打邊說明**：

```python
import random
import matplotlib.pyplot as plt

# 步驟 1：模擬一次擲骰子
def roll_dice():
    return random.randint(1, 6)

# 測試一次
print(roll_dice())
```

> 「我先寫最小的東西：一個骰子。先確定它能跑。」

```python
# 步驟 2：擲很多次，收集結果
N = 1000
results = []

for i in range(N):
    results.append(roll_dice())

print(f"共擲 {N} 次，前 10 個結果：{results[:10]}")
```

> 「現在我們有 1000 個數字，但這樣看不出什麼。需要視覺化。」

```python
# 步驟 3：畫直方圖
plt.figure(figsize=(8, 5))
plt.hist(results, bins=6, range=(0.5, 6.5), edgecolor='black')
plt.xlabel("骰子點數")
plt.ylabel("出現次數")
plt.title(f"擲骰子 {N} 次的結果分佈")
plt.xticks([1, 2, 3, 4, 5, 6])
plt.show()
```

**刻意製造一個錯誤**：把 `range=(0.5, 6.5)` 拿掉，讓學生看到 bins 分割不對的圖，再修回來。

> 「直方圖的 bins 怎麼設，會影響圖的外觀。這個細節之後會再提。」

**最後提問**：「如果把 N 從 1000 改成 10，圖會差很多嗎？我來改一下——」（實際改，讓學生看到差異）

### 學生實作（35 分鐘）

**任務**：完成骰子模擬程式，並回答以下問題（口頭或寫在程式注解裡）：

1. 跑 100 次 vs 10000 次，直方圖有什麼不同？
2. 如果把骰子換成 12 面骰（`random.randint(1, 12)`），需要改哪裡？
3. 試著讓標題自動顯示 N 的值（提示：用 f-string）

**老師巡視重點**：
- 確認每個學生的程式可以執行（先解決 import error）
- 對快的學生：「你能讓程式同時畫 N=100 和 N=10000 的對比圖嗎？」
- 對慢的學生：確認他們至少完成步驟 1 和 2，步驟 3 可以帶回家做

---

### 回顧與預告（20 分鐘）

**本週 3 個關鍵點**（寫在白板，學生截圖）：

1. 模擬 = 把現實簡化成模型，用電腦跑很多次
2. 一次實驗沒意義，要重複夠多次才能看規律
3. 圖表是「說故事」的工具，不只是輸出結果

**本週作業說明**（見 assignment-brief）

**下週預告**：「下週我們要用模擬來玩彩票——看看你要買幾張才可能中獎。」

---

### 常見問題與處理

| 問題 | 處理方式 |
|------|----------|
| `import matplotlib` 失敗 | 請學生在終端機跑 `pip install matplotlib`；若環境問題嚴重，先用 `print` 代替圖表 |
| `random.randint(1, 6)` 的範圍 | 強調 Python 的 randint 是包含兩端的（與某些語言不同） |
| 為什麼要用函數包起來 | 「之後要跑 1000 次，有函數才能用迴圈叫它」——先種下 simulate_once 的概念 |
| 直方圖的 bins 怎麼決定 | 本週先給固定答案（bins=6），下週會深入討論 |
