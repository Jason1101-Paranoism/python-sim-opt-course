# Level 3 週 10：1-on-1 討論指南

## 本週個別討論聚焦

本週是 Level 3 的最後一次 1-on-1，核心任務有三：

1. **展示後個別回饋**：針對期末展示給出具體的書面 + 口頭評語
2. **作品集完成確認**：確認程式碼整理、README、報告草稿的狀態
3. **下一步規劃**：Level 4 意願、競賽/申請計畫、未來延伸方向

---

## 建議討論流程（60 分鐘）

| 時段 | 時間 | 內容 |
|------|------|------|
| 展示後個別回饋 | 15 分鐘 | 針對今日期末展示給出具體評語；回答展示中沒答出的技術問題 |
| 作品集確認與指導 | 25 分鐘 | 查看程式碼整理狀態；讀 README；確認報告草稿品質 |
| 下一步規劃 | 15 分鐘 | Level 4 討論；競賽/申請計畫；未來延伸方向 |
| 完課確認 | 5 分鐘 | 確認所有完課條件達成；說明結業說明的時程 |

---

## 具體問題清單

### 展示後個別回饋段（15 分鐘）

**老師先給結構性回饋（5 分鐘）**：

說明以下四點（根據今日展示的觀察）：
1. 「你的問題陳述 — 清楚嗎？聽眾有沒有立刻理解你在做什麼？」
2. 「你的方法說明 — 技術細節夠深嗎？有說工具名稱？有說為什麼選這個工具？」
3. 「你的結果 — 數字夠具體嗎？有說結果為什麼合理？」
4. 「你的限制 — 夠具體嗎？說清楚了在什麼情境下可能偏差？」

**補答展示中沒答出的技術問題（10 分鐘）**：

老師選出今日問答中學生沒能回答的問題，在這裡一起討論和補充：

範例：
- 「你說不確定你的解是局部還是全局——我們現在一起驗證一下。你的目標函數圖長什麼樣？」
- 「你說的均勻混合假設——能不能說更具體一點？哪種現實情境最容易違反這個假設？」

---

### 作品集確認段（25 分鐘）

**問題一：程式碼整理狀態（10 分鐘）**

- 「讓我看你的主程式。你的函數有 docstring 嗎？」
- 「有沒有死碼（被注釋掉的舊程式）？」
- 「主程式可以從頭執行一次不報錯嗎？現在試一下。」

若程式碼還沒整理好：
- 一起花 5 分鐘整理最重要的一個函數（示範格式）
- 其他部分約定在 1 週內完成繳交

---

**問題二：README 品質確認（5 分鐘）**

讓學生打開他的 README，老師快速讀一遍，確認：
- 「一個不認識你的人，讀完這個 README，能不能知道你在做什麼？」
- 「執行方式有沒有說清楚？（pip install、python main.py 等具體指令）」
- 「主要結果有沒有一個數字？」

若 README 還不完整：
- 當場一起補充最重要的部分（通常是執行方法 + 主要結果）

---

**問題三：技術報告草稿（10 分鐘）**

讀學生的方法段和限制段（最重要的兩段）：

**方法段核查**：
- 「你的工具有說到函數名稱嗎？（不只說 SciPy）」
- 「你的步長設定有說為什麼嗎？（引用收斂測試）」
- 「你的最佳化方法選擇有說理由嗎？」

**限制段核查**：
- 「你有幾個限制？都夠具體嗎？」
- 「你有說每個限制的『影響方向』嗎？（往高估？往低估？）」

若報告草稿缺少段落：
- 說「這週要補完的是 [X 段]，這樣你的報告才有完整六個章節」

---

### 下一步規劃段（15 分鐘）

**問題一：Level 4 意願（5 分鐘）**

- 「你有沒有興趣繼續做 Level 4（研究型專題）？」
- 「Level 4 的重點是：文獻回顧 + 完整的研究論文格式。和 Level 3 相比，要讀論文、要說清楚你的工作和前人的差別在哪。你感興趣嗎？」
- 若有興趣：說明 Level 4 的先修條件（Level 3 完課）和大致方向

---

**問題二：競賽/申請計畫（5 分鐘）**

根據之前的策略備註，確認計畫：

- 若有科展計畫：「你的題目適合科展的格式嗎？科展需要實驗設計，你的數值模擬要怎麼轉換成科展的形式？」
- 若有大學申請計畫：「你需要英文摘要嗎？需要 PDF 報告嗎？截止日期是什麼時候？」
- 若沒有具體計畫：「你有沒有考慮把這份作品放到 GitHub 上？這是最低成本、但長期最有用的事。」

---

**問題三：這個專題的未來延伸（5 分鐘）**

一起想 1–2 個具體的延伸方向：

- 「你的模型最不完整的假設是 [X]。如果要改掉這個假設，你需要加什麼？」
- 「你的最佳化現在是一維的（只最佳化一個變數）。如果你要最佳化兩個或三個變數，你預計會遇到什麼新問題？」
- 「有沒有你做到一半、因為時間不夠而放棄的方向？」

---

### 完課確認段（5 分鐘）

**逐一確認完課條件**：

| 完課條件 | 狀態 | 備註 |
|---------|------|------|
| 出席期末展示 | ✓ | 今日完成 |
| 繳交主程式（可執行） | 待確認 | |
| 繳交方法段 + 實驗設計段草稿 | 待確認 | |
| 期末展示完成 | ✓ | 今日完成 |
| 作品集整理（1 週後） | 排程中 | |

若有未完成的條件，明確說明：「你需要在 [日期] 前補交 [X]，才能確認完課。」

**告知完課後流程**：
- 完課確認信會在作品集收到後發送
- 若要申請結業證書，請告知老師
- Level 4 招生時程（若學生有興趣）

## 本週學生應達成的里程碑

| 里程碑 | 驗收時間 |
|--------|---------|
| 期末展示完成 | 今日課堂 |
| 展示後個別回饋已收到 | 今日 1-on-1 |
| 作品集整理方向已確認 | 今日 1-on-1 |
| 下一步計畫（競賽/Level 4/GitHub）已確定 | 今日 1-on-1 |
| 完整作品集繳交 | 1 週後 |

## Level 3 最後一次 1-on-1 的老師自我提醒

1. **誠實的最終評語**：不要因為是最後一次就說什麼都好。若有明顯技術缺失，清楚告知，並說明「這個問題在 Level 4 要解決」。

2. **肯定具體的成長**：找出這位學生在 Level 3 中最明顯的技術進步（從週 1 到週 10），具體說出來。

3. **記錄在學生紀錄表**：填寫完整的 Level 3 完課紀錄，包括：主要技術成就、未解決的問題、Level 4 建議、競賽/申請計畫。

4. **拍照備份**：若學生同意，可保留一份他的展示簡報截圖（不含個資）作為課程案例。
